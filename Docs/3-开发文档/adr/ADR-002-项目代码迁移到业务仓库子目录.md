# ADR-002 · 项目代码迁移到 {{APP_REPO_DIR}}/ 子目录

- **状态**：现行
- **日期**：2026-05-08
- **决策人**：zlbdh
- **关联**：—（结构性调整，不对应 PRD 功能）

> ⚠️ **当前覆盖说明（2026-06-15）**：`{{APP_REPO_DIR}}/` 作为业务代码仓库仍是现行结构；本文中的 `rules/` 目录是 2026-05 当时口径，后续已由 ADR-007、ADR-029 演进为 `操作系统/` + `能力资产/`。

## 背景

2026-05-08 之前，「{{PROJECT_NAME}}」项目根 `D:\\WGKJ\\{{PROJECT_NAME}}\\` 是**单层平铺**结构：22 个东西全在一级——代码（src/、public/、android/、dist/、node_modules/）+ 9 个根级配置文件 + 规范（rules/）+ 文档（Docs/）+ 改动池（确认改动/）+ APK 归档（apk/）+ CI（.github/）+ 入口（README.md）。

zlbdh 看了文件浏览器后明确表示"这么乱"，需要把项目代码独立出来。

候选方案：

| 选项 | 描述 | 评估 |
|---|---|---|
| 不动 | 接受 22 项一级 | 找东西每次翻 |
| 迁到 `code/` | 通用名 | 中性 |
| **迁到 `{{APP_REPO_DIR}}/`** | 跟最早设计一致，命名贴合项目实例 | 选这个 |
| 迁到 `app/` | 短，但跟 `apk/` 都是 a 开头 | 略糊 |

## 决定

**代码 + 配置全部迁入 `{{APP_REPO_DIR}}/` 子目录**，根目录只留文档、规则、改动池、APK 归档和入口说明：

```
{{PROJECT_NAME}}/
├── {{APP_REPO_DIR}}/             ← 项目代码
├── rules/            ← AI 规范
├── Docs/             ← 文档
├── 确认改动/         ← 改动池
├── apk/              ← APK 归档
└── README.md         ← 入口
```

**迁入 `{{APP_REPO_DIR}}/` 的 14 项**：
- 5 个目录：`android/`、`public/`、`src/`、`dist/`、`node_modules/`
- 9 个配置文件：`.gitignore`、`package.json`、`package-lock.json`、`vite.config.js`、`tailwind.config.js`、`postcss.config.js`、`capacitor.config.json`、`index.html`、`build-apk.bat`

**保留在根的**：
- `rules/`、`Docs/`、`确认改动/`、`apk/`、`README.md`

2026-05-08 后续修正：当前远端 Git 仓库根目录就是 `{{APP_REPO_DIR}}/`，因此 GitHub Actions 必须放在 `{{APP_REPO_DIR}}/.github/workflows/`，否则不会随 `zlbdh/{{APP_REPO_DIR}}` 推送生效。

## 后果

### 好处
- 根目录从 22 项缩到 6 项左右，**职责一目了然**
- 规范（`rules/`）和代码（`{{APP_REPO_DIR}}/`）独立，AI 工作时不会混淆
- 这套结构可作为模板**复制到其他项目**（每个项目都按 `项目名/`+`rules/Docs/{{APP_REPO_DIR}}/...` 组织）

### 代价
- 所有 npm/npx 命令前要 `cd {{APP_REPO_DIR}}/`：
  ```cmd
  cd /d D:\\WGKJ\\{{PROJECT_NAME}}\\{{APP_REPO_DIR}}
  npm install / npm run dev / npm run build / npm test
  ```
- `build-apk.bat` 因 `cd /d "%~dp0"` 自动定位脚本所在目录，**内部相对路径全部仍有效**，无需改逻辑（仅更新提示文案）
- GitHub Actions：放在 `{{APP_REPO_DIR}}/.github/workflows/build-apk.yml`，workflow 内路径以 `{{APP_REPO_DIR}}/` 为仓库根，不再使用 `working-directory: ./{{APP_REPO_DIR}}`
- Docs/Agent 里"`src/...`"约定为"以 `{{APP_REPO_DIR}}/` 为基准的相对路径"，无需逐处加前缀
- README.md 项目结构图整体重画

### 后续如果反悔了
- **触发条件**：{{APP_REPO_DIR}} 这层让结构反而更难找（比如几乎所有命令都要 cd 进去）
- **撤销路径**：`mv {{APP_REPO_DIR}}/* .` + `rmdir {{APP_REPO_DIR}}` + 反向更新 `{{APP_REPO_DIR}}/.github/workflows/build-apk.yml` + 重写 README.md 项目结构图 + 在所有运维文档里把 `{{APP_REPO_DIR}}\\` 前缀去掉
- **数据迁移**：无（纯文件移动 + 文档路径文本替换）
- **预估翻案成本**：30 分钟

## 备注

- 迁移过程中触发了一次 mount 9KB 截断事故：`Sprint-1需求清单.md` 和 `需求历史.md` 在 Edit 后被截断，已用 Python 重写恢复。**已写进经验**：>8KB 文件不能用 Edit 工具，必须用 Python 写入。这条建议同步进 `rules/code/已知技术约束.md` 的"Mount 写入限制"段。
- 这次迁移可作为 RETRO 第 1 次的参考案例（"L4 改动如何在 1 次会话内完成"）。
