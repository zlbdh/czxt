# ADR-004 · 大文件拆分（PROP-001 B3 全拆方案实施）

- **状态**：现行
- **日期**：2026-05-08
- **决策人**：zlbdh（PROP-001 审批选 B3）
- **关联**：[PROP-001-2026-05-08-拆分大文件.md](../../../确认改动/已审批/已完成/PROP-001-2026-05-08-拆分大文件.md)

## 背景

2026-05-08 健康检查发现 5 个源文件超过 / 临界 9KB mount 写入限制：

| 文件 | 原大小 |
|---|---:|
| `{{APP_REPO_DIR}}/src/features/home/Home.jsx` | 13208B |
| `{{APP_REPO_DIR}}/src/shared/components.jsx` | 12137B |
| `{{APP_REPO_DIR}}/src/features/timeline/Timeline.jsx` | 9594B |
| `{{APP_REPO_DIR}}/src/features/health/Health.jsx` | 9133B |
| `{{APP_REPO_DIR}}/src/features/profile/Profile.jsx` | 8579B |

每次改这些文件都必须用 Python 写入，Edit/Write 触发 mount 截断（已发生 2 次实际事故，丢字节）。

## 决定

按 PROP-001 的 B3 方案**全部拆分**，让每个源文件 < 6KB（Edit/Write 安全区）。

### 拆分映射

#### components.jsx (12137B → 6 个文件)
- `components.jsx` (核心 Card/PageHeader/SectionTitle/EmptyState) — 2017B
- `buttons.jsx` (PillButton/PrimaryButton/GhostButton) — 1023B
- `inputs.jsx` (TextInput/TextArea) — 501B
- `stats.jsx` (StatRow/Stat) — 1460B
- `MonthCalendar.jsx` (独立) — 4075B
- `TaskDetailModal.jsx` (独立) — 3284B

#### Home.jsx (13208B → 6 个文件)
- `Home.jsx` (主组合 + state + handlers + Training/Habits/Modal 内嵌) — 6673B
- `HomeStatBar.jsx` (Today's Pulse) — 928B
- `HomeTaskSections.jsx` (4 时段任务清单) — 3222B
- `HomeHydration.jsx` (喝水卡) — 1542B
- `HomeAIBriefing.jsx` (AI 简报) — 884B
- `HomeChat.jsx` (聊天) — 1906B

#### Timeline.jsx (9594B → 2 个文件)
- `Timeline.jsx` (主 + 记录新碎片 + AI 整理 + 月历) — 6279B
- `TimelineList.jsx` (时间线 Card) — 3849B

#### Health.jsx (9133B → 2 个文件)
- `Health.jsx` (主 + 体重 + 训练 + 月历) — 5955B
- `HealthMeals.jsx` (饮食 Card) — 4319B

#### Profile.jsx (8579B → 2 个文件)
- `Profile.jsx` (主 + 热力图 + 身体档案) — 3684B
- `ProfileExtras.jsx` (AI/备份/设备/最近问题 4 卡) — 5444B

#### import 引用更新
5 个 features 文件的 `from '../../shared/components.jsx'` 拆分为按需细分的 import（每个文件 4-5 行 import 语句）。

## 后果

### 好处
- **5 个高危源文件全部进入安全区**（最大也只 6.7KB，离 9KB 限制远）
- 咪咪以后可以重新自由用 Edit 工具改这些组件
- 单文件可读性大幅提升（一屏能看完）
- 测试可以更细颗粒度（per-component）

### 代价
- 文件数量从 ~12 增加到 ~26（13 个新文件）
- import 语句从 1 行变 4-5 行（小幅冗余）
- props 传递层级增加 1 层（HomeStatBar 等子组件需通过 prop 拿数据）

### 后续如果反悔了
- **触发条件**：拆分粒度让查找代码更难（不太可能，因为命名清晰）
- **撤销路径**：手动合并回去，约 1 小时
- **预估翻案成本**：低
