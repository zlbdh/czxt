# ADR-006 · 导航重构：5 Tab 重排 + 聊天独立化 + 「我」上提到右上角

- **状态**：现行（zlbdh 2026-05-09 批 PROP-003 后实施）
- **日期**：2026-05-09
- **决策人**：zlbdh / 咪咪
- **关联**：[PROP-003](../../../确认改动/已审批/已完成/PROP-003-2026-05-09-导航重构与聊天独立化.md) / F-NAV-1 / F-CHAT-1

> ⚠️ **当前覆盖说明（2026-06-15）**：本文的底部 Tab 与 `/profile` 信息架构仍可作为 PROP-003 历史依据；实际路由以 `{{APP_REPO_DIR}}/src/app/AppShell.jsx` 为准。文中 “API key” 是设置入口描述，不代表真实密钥可外传或写入 tracked 文件。

## 背景

v2.3 dev APK 真机使用后发现两条信息架构问题：

1. **「我」占着 1/5 底部 Tab，但用得少** — 设置类入口（资料、API key、备份、热力图）一周用不了几次
2. **聊天埋得深** — 「聊聊」是高频功能（咪咪 AI 对话），但目前是 Home 页中段的一张卡，每次要进首页 → 滑到中段 → 点开 → 输入，4 步才能聊

底部 Tab 是寸土寸金，应该全是高频主功能；设置类入口应通过永久可达的角落 affordance 触达。

候选方案（PROP-003 已枚举）：
- B1 不动 → 否决（痛点持续）
- B2 只移「我」不加「聊天」→ 否决（4 Tab 稀疏 + 聊天入口仍深）
- B3 加「聊天」不移「我」→ 否决（6 Tab 违反移动端 5 Tab 上限）
- **B4 移「我」+ 加「聊天」**（5 Tab 重排 + 右上角永久头像入口） → **采纳**

## 决定

### 信息架构

| 位置 | 旧 | 新 |
|---|---|---|
| Tab 1 | 今日 (/) | 今日 (/) |
| Tab 2 | 健康 (/health) | 健康 (/health) |
| Tab 3 | 记账 (/accounting) | **聊天 (/chat)** ★ |
| Tab 4 | 时光 (/timeline) | 记账 (/accounting) |
| Tab 5 | 我 (/profile) | 时光 (/timeline) |
| 右上角 | — | **永久 30×30 头像 → /profile** ★ |

聊天放第 3（中间位）— 移动端拇指自然热区，符合"高频功能放中央"的规律。

### 代码层

1. **新建 `{{APP_REPO_DIR}}/src/features/chat/Chat.jsx`**（全屏聊天页）
   - 把 `{{APP_REPO_DIR}}/src/features/home/HomeChat.jsx` 的所有交互平移过来：消息列表 + 输入框 + 发送按钮 + Enter 提交 + loading dots
   - 新增：自动滚到底（每条新消息后 scrollTo）
   - 新增：F-203 归档抽屉直接挂这里（点 SectionTitle 的 `查看历史` action 打开）
   - 复用：`addChatMessage` action / `chatArchive.js` 纯函数 / `ChatHistoryDrawer.jsx`

2. **改 `{{APP_REPO_DIR}}/src/app/AppShell.jsx`**
   - tabs 数组：5 项重排（Home / Health / **Chat** / Accounting / Timeline）
   - Profile 路由保留，**不进 tabs 数组**（不显示在底部）
   - 5 个 ErrorBoundary 各自 key（含新的 `key="chat"`）
   - `import ChatPage from '../features/chat/Chat.jsx'`

3. **改 `{{APP_REPO_DIR}}/src/shared/components.jsx` 的 `PageHeader`**
   - 把现有 `action` prop 升级为 `rightAction`（语义更明确）
   - **default**：当 `rightAction` 未传时，自动渲染 30×30 圆形 User icon button → `Link to="/profile"`
   - 边框：1px solid `inkSoft` (#3a3128)
   - **`rightAction={null}`**：显式传 null 表示不渲染（Profile 页用此关掉，避免 self-link）

4. **改 `{{APP_REPO_DIR}}/src/features/home/Home.jsx`**
   - 移除 `import HomeChat from './HomeChat.jsx';`
   - 移除 `<HomeChat ... />` JSX
   - 移除 `chatText / chatLoading / sendChat / recentChat` state + 函数
   - 移除 `callAI` import（已无引用）
   - Home 自定义 hero header（3.2rem 周X）增加右上角头像（与 PageHeader 风格保持一致，使用同一辅助组件）

5. **删除 `{{APP_REPO_DIR}}/src/features/home/HomeChat.jsx`**（无引用，按 ADR-006 不留废文件）

### 视觉规范（新增到 `rules/visual/视觉设计规范.md`）

**右上角入口 affordance**
- 位置：每页 PageHeader 右侧，与标题底对齐
- 形状：30×30 圆形（`rounded-full`）
- 描边：1px solid `inkSoft` (#3a3128)
- 内容：`User` 图标（lucide-react，14×14，`color: ink`），居中
- 交互：点击 → `<Link to="/profile">`
- 例外：Profile 页本身传 `rightAction={null}` 隐藏

## 后果

### 好处
- 聊天从 4 步触达 → 1 步触达（点 Tab 即开）
- 信息架构跟使用频率对齐（底部全是高频，设置走右上角）
- F-203 的归档抽屉有了语义合理的家（不是 Home 卡片，而是 Chat 主页）
- Home.jsx 字节数下降（HomeChat 移除）

### 代价
- PageHeader 接口调整（`action` → `rightAction`），影响 5 个 feature 头部，但向下兼容（旧的 `action` 仍接受，但优先级低于 `rightAction`）
- 用户已经习惯了「我」在底部右下角，需要 1-2 次重新适应
- HomeChat.jsx 删除是不可逆的，但 git history 保留

### 后续如果反悔了
- **撤销路径**：把 Chat.jsx 内容拆回 HomeChat.jsx，AppShell tabs 数组 + ErrorBoundary 还原 5 项
- **预估翻案成本**：1.5h（含改 4 处文档）
- **数据无损**：聊天表 chatMessages 完全不动
