# ADR-009 · 框架自动化 v2 边际优化（PROP-006）

## 背景

ADR-008（PROP-005）把规范从「半自动」推到「90% 全自动」。v20 体检发现 6 个**边际改进点**（90% → 95% 的空间），不是阻塞性风险但是真隐患：

1. 状态.md 没失效检测 → AI 可能用过期缓存
2. 状态推断「建议」无追踪 → 重复推荐已 dismiss 的事
3. DoD 越来越长 → AI 选择性忽略
4. 「session 起手」靠 AI 自觉 → 没强制触发

#5 #6（PROP 卡进行中告警 / 占位主动监控）作为低优先级，留 RETRO-003。

## 决策

4 项边际改进：

### P1 — 状态.md stale 检测
状态推断 推断 5 升级：状态.md mtime vs 最近代码改动 mtime，差 ≥ 1 天 → 标 🟡 stale。stale 状态下 AI **不直接用 状态.md 数据**，主动跑推断 1-4 重新对账。

### P2 — 建议-决议追踪
状态.md 加「最近建议 + 用户决议」表段 + 状态推断 skill 加「防重复推荐机制」段。已 dismiss 建议 7 天内不再重推。

### P3 — DoD 重组
实施循环.md L3/L4 DoD 顶部加「⭐ 核心 5 项必勾」摘要：
- 代码 / 测试 / PRD / PROP 收档 / 跑两个对账 skill
其余项标记为「扩展」，按需补。

### P4 — Session 起手强制触发
两处显眼提醒：
- agent/INDEX.md 顶部加 🚨「新会话第一动作」4 步顺序
- 项目根 README.md 加同样段，让 AI 一进项目根就看到

## 量化指标（90% → 95%）

| 指标 | ADR-008 后 | ADR-009 后 |
|---|---|---|
| 状态.md 过期使用率 | 高 | ≤ 0（stale 自动检测） |
| 重复推荐已 dismiss 建议 | 频繁 | 0（7 天 cooldown） |
| DoD 项漏勾率 | 中 | 低（核心 5 项强制） |
| Session 起手「忘了跑推断」 | 中 | 0（README + INDEX 顶部硬提醒） |
