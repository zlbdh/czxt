# ADR-012 · 交接卡机制（PROP-009）

## 背景

2026-05-11 Claude Code 写代码 → 切到 Codex 跑测试场景，zlbdh 指出：「问题不是手动复制粘贴，而是状态及信息的流转」。

**真实痛点**：当前同步包 3 段（完成清单 / 下一步 / 警戒）太抽象——接手者要花 5-10 分钟翻 git status / wc -c / 推断历史才能续上。

具体场景：
- Claude Code 写完代码 → Codex 不知道改了哪些文件、字节多少、跑了什么测试、还有什么没跑
- Codex 测完 → Claude Code 不知道哪些测试通过、哪些失败、有没有回炉代码

## 决策

把同步包升级为**结构化「交接卡 6 段」**：

### 6 段必填
1. **时间 + 角色切换** — 从 A 到 B
2. **文件变更清单** — `+ M -` 标记 + 字节数 + ≥6500B 警告
3. **测试状态表** — 端能力对照（哪端跑了什么、结果）
4. **接手者待办** — 具体 checklist（不接受「测一下」）
5. **警戒事项** — 大文件 / TODO / 风险
6. **追加问答** — 接手者回炉问题时双向

### 双层防御
1. **项目根 `状态.md` 顶部「最近交接卡」** — 跨 session 缓存，完整 6 段
2. **简版交接卡文本** — 用户复制粘贴用，6 段精简

## 量化指标

| 指标 | PROP-009 前 | PROP-009 后 |
|---|---|---|
| 跨工具切换接手时间 | 5-10 分钟 | **≤ 1 分钟** |
| 信息丢失率 | 中（接手者要推断）| 低（结构化必填）|
| 大文件触发漏报 | 偶发 | 几乎不会（必填段）|
| 测试状态对账 | 模糊 | 端能力对照表 |

## 替代关系

ADR-012 **不替代任何前 ADR**——它是 PROP-005 → 006 → 007 自动化升级路径的延续。
- 继承 ADR-008 的「跨 session 同步包」概念，但格式更严
- 继承 ADR-010 的 6500B 硬规则，把警告内嵌到交接卡 ② 段
- 继承 ADR-007 的 5 大分组结构

## 向后兼容

旧 3 段同步包格式（完成 / 下一步 / 警戒）**全部内嵌到新 6 段**：
- 旧「完成」≈ 新 ① + ②
- 旧「下一步」≈ 新 ④
- 旧「警戒」≈ 新 ⑤
- 新增 ③（测试状态）+ ⑥（追加问答）

读到旧格式时，状态推断 推断 5 提示「建议下次实施时升级到 6 段」，不强制改历史。

## 后果

### 收益
- ✅ 跨工具接手时间从 5-10 分钟 → ≤ 1 分钟
- ✅ Claude Code / Codex / Cowork 三端协作有统一格式
- ✅ 大文件警告内嵌交接卡 ② 段，不再漏报
- ✅ 测试状态有端能力对照表，「假装跑了」可识别

### 代价
- ❌ 状态.md 顶部段更长（约 1.5KB 增加）
- ❌ 写交接卡比写 3 段同步包多 30 秒（但接手者节省 5-10 分钟，净赚）
