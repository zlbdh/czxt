# ADR-013 · 交接卡格式独立拆出 — 自纠 PROP-009 副作用

- **状态**：现行
- **日期**：2026-05-11
- **决策人**：zlbdh / Claude Code 发现
- **关联**：[PROP-010](../../../确认改动/已审批/已完成/PROP-010-2026-05-11-交接卡格式独立拆出.md) · 自纠 [ADR-012](ADR-012-交接卡机制.md) 副作用 · 遵循 [ADR-010](ADR-010-框架自动化v3-9隐患全封堵.md) P1-3

> ⚠️ **当前覆盖说明（2026-06-15）**：本文保留 2026-05 `agent/workflows/交接卡格式.md` 拆分原貌。当前交接卡真源为 [`../../../操作系统/03_交接/交接卡格式.md`](../../../操作系统/03_交接/交接卡格式.md)，chat 简版已扩为 ①-⑦。

## 背景

PROP-009 / ADR-012 把同步包升级为「交接卡 6 段」格式，写入 `agent/workflows/实施循环-DoD.md`。但加入完整模板后，文件涨到 **7029B**，越 ADR-010 P1-3 硬门槛（≥6500B 必拆）。

Claude Code 在 PROP-008 收档时发现此越线，记入交接卡 ⑤ 警戒，建议新开 PROP-010 拆。

**讽刺**：刚立完「6500B 必拆」规则就自己破。**这恰好证明规则有效** — 触发条件自动响应。

## 决策

把交接卡完整模板从 `实施循环-DoD.md` 抽出到独立文件 `agent/workflows/交接卡格式.md`。

`实施循环-DoD.md` 改成短引用：
- 速记两层防御要点
- 详见 `交接卡格式.md` 完整模板

## 实施

| 文件 | 改动 | 字节变化 |
|---|---|---|
| `agent/workflows/交接卡格式.md` | 新建 | 0 → 3409B |
| `agent/workflows/实施循环-DoD.md` | 抽出交接卡段 + 改短引用 | 7029B → ~5800B |

实施后 ≥6500B 文件 = 0 ✅

## 后果

### 收益
- ✅ 实施循环-DoD.md 回到安全区
- ✅ 交接卡格式独立维护，未来扩展不阻塞 DoD
- ✅ 自纠机制有效——PROP-009 副作用立刻被 ADR-010 P1-3 + Claude Code 体检 + 交接卡 ⑤ 警戒**三重检测**

### 教训
- 立完规则后**第一个实施动作就应该检查自己是否越线**
- DoD 跑项目体检 + 状态推断 应该**包含本次实施引入的新文件**

### 不可预防的
- 任何新机制（如交接卡 6 段）落到文件里就有字节增长
- 关键是规则触发 + 响应足够快（这次 < 1 天内自纠 ✓）
