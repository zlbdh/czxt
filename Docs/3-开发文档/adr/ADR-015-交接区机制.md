# ADR-015 · 交接区机制（PROP-011）

- **状态**：现行
- **日期**：2026-05-11
- **决策人**：zlbdh
- **关联**：[PROP-011](../../../确认改动/已审批/已完成/PROP-011-2026-05-11-交接区机制.md) · 接续 [ADR-012](ADR-012-交接卡机制.md) / [ADR-013](ADR-013-交接卡格式独立拆出.md)

> ⚠️ **当前覆盖说明（2026-06-15）**：本文保留 2026-05 交接区机制初版和 `agent/workflows` 旧路径原貌。当前交接目录规则以 [`../../../交接区/README.md`](../../../交接区/README.md) 为准，完整卡格式以 [`../../../操作系统/03_交接/交接卡格式.md`](../../../操作系统/03_交接/交接卡格式.md) 为准。

## 背景

PROP-009 / ADR-012 引入「交接卡 6 段」格式，写在 `状态.md` 顶部段。PROP-006 跨 session 同步包从 3 段升级到 6 段。但实战发现：

| 痛点 | 严重 |
|---|---|
| 只能存 1 张交接卡（顶部段覆写丢历史）| 🔴 |
| 多 session 同时改 状态.md 易冲突 | 🟡 |
| 状态.md 既是真相又是缓存职责混合 | 🟡 |
| 接手者只能读「最新」（被覆盖的丢失）| 🔴 |

zlbdh 2026-05-11 提出**交接区**方案：每张卡独立文件 + 待接手 / 已接手 / 历史归档 三态目录。

## 决策

新建顶层 `交接区/` 目录，把交接卡从「状态.md 单点覆写」升级为「独立文件事件流」。

### 目录结构
```
交接区/
├── README.md
├── 待接手/                ← 工具完成实施后写这里
├── 已接手/                ← 接手者完成实施后 mv 这里（永久存档）
└── 历史归档/              ← 30 天以上归档（按月分目录）
```

### 文件命名
`YYYY-MM-DD-HHMM-任务-从角色到角色.md`

例：`2026-05-11-1435-F-006-Codex到zlbdh.md`

### mv 责任
**接手者完成实施时 mv「自己接的那张」**从 `待接手/` 到 `已接手/`，同时写新卡到 `待接手/`。
zlbdh confirm 时，AI 帮把「给 zlbdh 的卡」mv 到 `已接手/`。

### 状态.md 角色变化
- 旧（PROP-009）：顶部「最近交接卡 6 段」详细完整
- 新（PROP-011）：顶部「最近交接卡摘要 + 链接到 交接区/待接手/最新」精简
- 详情 → 交接区/独立文件
- 状态.md 回归「单纯快照 + 进度速查 + 推断决议」职责

## 三层结构最终图

```
交接区/待接手/2026-XX-XX-HHMM-任务-从到.md  ← 完整 6 段独立文件（事件流）
        ↓ 链接
状态.md 顶部摘要段                          ← 极简快照
        ↓ 复制
简版交接卡文本                              ← 给用户贴另一端 session
```

## 量化收益

| 指标 | PROP-009 | PROP-011 |
|---|---|---|
| 历史可追溯 | ❌ 覆写 | ✅ 每张独立文件 |
| 多 session 防冲突 | ⚠️ 同改易冲突 | ✅ 不同文件天然防 |
| 接手「读哪张」 | 只能读最新（旧丢失）| ✅ 按时间 / 按任务挑 |
| 状态.md 职责 | 真相 + 缓存混合 | 单纯快照 |
| 接手时间 | ≤ 1 min（PROP-009 已优化）| ≤ 30s（更明确） |

## 替代关系

ADR-015 **不替代 ADR-012**，是 ADR-012 的**存储位置升级**：
- ADR-012「交接卡 6 段格式」保留
- 写入位置从 状态.md 顶部段 → 交接区/独立文件
- 状态.md 顶部段不删，改成摘要 + 链接

## 后果

### 收益
- ✅ 历史可追溯（git log + 文件名时间戳）
- ✅ 多 session 天然防冲突（不同文件）
- ✅ 状态.md 职责单一
- ✅ 接手者明确读哪张

### 代价
- ❌ 多一个顶层目录（认知负担）
- ❌ mv 操作责任要规范（已在 交接区/README.md 写清）
- ❌ 老 session 习惯改读 状态.md 顶部段，需要适应

### 风险（已防）
- ✅ mv 责任明确：「接手者完成时 mv 自己接的那张」
- ✅ 多卡积压检测：状态推断 推断 5 加 卡数 ≥ 3 告警
- ✅ 历史档案：已接手 / 历史归档永久保留，不删
