# ADR-020 · 架构债清理：4 文件拆分 + 2 处死代码删除（PROP-016）

- **状态**：现行
- **日期**：2026-05-12
- **决策人**：zlbdh
- **关联**：[PROP-016](../../../确认改动/已审批/已完成/PROP-016-2026-05-12-架构债清理.md) · RETRO-007 · Sprint-4 业务前置阻塞解除

## 背景

Sprint-3 收官后，业务已稳定到 v3.5，但代码层累积了 4 个接近警戒区的文件和 2 处死代码：

- `timelineBuilder.js` 6918B：今日时间轴逻辑 + utility 混在一起
- `Health.jsx` 6331B：健康页主容器继续承载体重趋势 JSX
- `dailyBriefing.test.js` 6272B：F-CHAT-4 测试集中在单文件
- `defaults.js` 6069B：仍保留已废弃 `TASK_HINTS`
- `TaskDetailModal.jsx`：F-DAY-1 切到 DayDrawer `taskDetail` 后已无业务入口

如果直接进入 Sprint-4，每次扩这些文件都会先遇到文件体量噪音，影响实施速度。PROP-016 的目标是先清债，再开业务。

## 决定

从 v3.5.1 起，执行一次纯重构：

1. `timelineBuilder.js` 保留 `buildWorkdayTimeline` / `buildWeekendTimeline` / `buildTodayTimeline` 业务入口，把无业务 utility 拆到 `timelineBuilderUtils.js`，并在主文件 re-export 原有 utility，保证旧 import 不破。
2. `Health.jsx` 只保留页面状态与编排，把体重趋势卡片抽成 `HealthWeightTrend.jsx`。不抽 hook，避免把和页面状态强耦合的 `weight/bodyFat/chartData/actions` 变得更绕。
3. `dailyBriefing.test.js` 按功能段拆分：主文件保留 greeting / stats / 总入口；`dailyBriefing.suggestions.test.js` 承接 suggestions / warnings。
4. 删除 `TaskDetailModal.jsx`，删除 `defaults.js` 里的 `TASK_HINTS`，以 `taskDetails.js` + DayDrawer `taskDetail` 作为任务详情唯一实现。
5. 本次不改 schema、不改业务逻辑、不引依赖、不改 UI 视觉目标；commit 使用 `refactor:` 前缀。

## 关键取舍

### 1. 接受 timelineBuilder / Health 略高于 5KB 目标

PROP-016 初始目标是单文件 < 5KB。实际结果：

| 文件 | Before | After | 说明 |
|---|---:|---:|---|
| `timelineBuilder.js` | 6918B | 5897B | 已脱离警戒区，继续拆会让核心 builder 断裂 |
| `Health.jsx` | 6331B | 5426B | 已脱离警戒区，继续抽 BMI / 训练段收益不大 |

最终接受 5.3-5.8KB：可读性优先于机械追 5KB，且距离 6500B 警戒线已有余量。

### 2. 测试 fixture 暂时重复

`dailyBriefing.test.js` 和 `dailyBriefing.suggestions.test.js` 各自保留同名 `FULL_DATA`，避免为测试 helper 再新增一个跨文件 fixture。后续如果简报测试继续扩张，再抽 `dailyBriefing.fixtures.js`。

### 3. HealthWeightTrend 选组件，不选 hook

本次目标是缩小 `Health.jsx` 的 JSX 体量。体重输入状态、图表数据和 `actions.addRecord` 仍由页面主组件掌控；抽 hook 会把状态更新路径拉长，得不偿失。

## 后果

### 收益

- 4 个警戒文件全部脱离区：`timelineBuilder.js` -15%，`Health.jsx` -14%，`dailyBriefing.test.js` -30%，`defaults.js` -33%。
- Sprint-4 开始前源文件体量更轻，Codex / Claude Code / Cowork 都更容易读和改。
- 删除 `TaskDetailModal.jsx` + `TASK_HINTS` 后，任务详情实现只剩 DayDrawer `taskDetail` 一条路径。
- vitest 仍为 399/399，测试数量严格不减；真机 smoke 覆盖 Sprint-3 5 个功能回归。

### 代价

- 模块数增加 2 个左右，vite build modules 从 2284 到 2286，属于可接受范围。
- `timelineBuilder.js` 与 `Health.jsx` 未完全压到 5KB 以下，未来若继续扩仍要先看上下文。
- `dailyBriefing` 测试存在少量 fixture 重复，换来测试文件之间低耦合。

### 风险与缓解

- import 断裂风险：`timelineBuilder.js` re-export utility，现有 import 链保持兼容。
- 死代码删除风险：grep + esbuild + APK smoke 均未见 `TaskDetailModal` / `TASK_HINTS` 运行时报错。
- 业务回归风险：v3.5.1 真机 smoke 覆盖 Home 时间轴、DayDrawer 双模式、Health 体重趋势与驼背、Profile 作息、Chat 今日简报。

## 后续

- Sprint-4 可以在 PROP-016 清债后启动业务 PRD。
- 如果继续扩 `dailyBriefing`，优先拆 fixture 或把主动建议从规则式推进到 LLM 生成。
- 如果继续扩 Health，优先拆训练记录或 HealthCalendars 周边，而不是回填到 `Health.jsx`。
