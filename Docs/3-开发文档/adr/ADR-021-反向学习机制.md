# ADR-021 · 反向学习机制：AGENTS.md + check-operating-system.ps1 + agent/CHANGELOG.md（PROP-017）

- **状态**：现行
- **日期**：2026-05-12
- **决策人**：zlbdh
- **关联**：[PROP-017](../../../确认改动/已审批/进行中/PROP-017-2026-05-12-agent健康度治理.md) · RETRO-007 元洞察「framework 是产品要像产品管」· 姐妹项目「账号管理 / srm 服务器管理」反向学习

> ⚠️ **当前覆盖说明（2026-06-15）**：本文记录反向学习机制的引入。当前脚本入口已迁至 [`能力资产/tools/scripts/check-operating-system.ps1`](../../../能力资产/tools/scripts/check-operating-system.ps1)，体检范围已从 P4a-P4d 扩展到 P4a-P4q；轻量日志真源为 [`操作系统/00_变更记录/CHANGELOG.md`](../../../操作系统/00_变更记录/CHANGELOG.md)。

## 背景

Sprint-3 收尾 + PROP-016 架构债清理后，元洞察浮现：framework 自身（`agent/` + `Docs/` + `确认改动/`）也应像业务代码一样被管理 —— 需要"版本演进日志"、"健康检查"、"快速入门"。

但{{PROJECT_NAME}}的 framework 缺三件配套机制：

1. **新工具加入项目时的快速入门**：`agent/INDEX.md` 是技术参考（5 大语义分组），但缺"5 秒上手"入口。新 AI 工具进来后要扫 agent/ 整套（~15 分钟）才能开始干活。

2. **跨 session 健康度自动监控**：当前依赖 zlbdh 主动「@咪咪 跑下项目体检」。AI 在 mount 缓存陷阱下手动判断，决策点容易踩坑（RETRO-005 卡 2 / PM 自纠 #9/#12 — bash `wc -c` 走 Cowork mount 返回陈旧缓存导致幻警戒）。

3. **轻量演进日志**：当前 framework 小改动（命名修复、引用修复、注释优化）没合适承接 —— 走 PROP 过重，不记又丢痕迹。

恰好 zlbdh 自有的另两个项目（账号管理 / srm 服务器管理）已有这三件成熟机制。PROP-017 决定**反向学习**：把姐妹项目的机制吸收进来，定制到{{PROJECT_NAME}}的语境。

## 决定

引入 3 个 framework 元规则机制（**必须配套引入，不可单引**）：

### 1. 项目根 `AGENTS.md`（5 秒入口）

新建 `{{PROJECT_ROOT}}\AGENTS.md`（90 行 / 3488B），覆盖起手 5 步、敏感操作必查清单（C 类边界）、文件大小规则（PROP-013 / ADR-017）、跨工具协作（chat 简版 ①-⑥ PROP-014 / ADR-018）、流程优先三阶判定（🔴 / 🟡 / 🟢）。

**铁律**：「极简入口」—— 详细规则永远在 `agent/` 内部，本文件只指路。修改前必问：新工具能不能 5 秒看完？

### 2. `tools/check-operating-system.ps1`（自动化健康检查，历史入口）

新建 PowerShell 5.1+ 兼容脚本（约 290 行），当时实现 4 阶段；当前已迁移并扩展：

| 阶段 | 检测内容 |
|---|---|
| P4a | 基础完整性（61 必查项：含 AGENTS.md / agent/CHANGELOG.md / 项目体检-检查项-7-8.md 等）|
| P4b | 字节大小（按工具分层 + 文件清单展开 + 颜色标记）|
| P4c | agent/ 引用频率（≥10 高 / 5-9 中 / 2-4 低 / 1 极低 / 0 死代码）|
| P4d | 状态.md 30 天新鲜度（mtime + 内容日期戳双校验）|

当前用法：`powershell -NoProfile -ExecutionPolicy Bypass -File 能力资产/tools/scripts/check-operating-system.ps1`。

### 3. `agent/CHANGELOG.md`（第 4 档案，历史入口）

当前轻量演进日志真源是 `操作系统/00_变更记录/CHANGELOG.md`。本文保留当时 `agent/CHANGELOG.md` 命名背景。

| 改动类型 | 主档案 |
|---|---|
| L3+ 架构 / 元规则演进 | PROP + ADR + CHANGELOG（仅索引行）|
| Sprint 复盘 | RETRO + CHANGELOG（仅索引行）|
| **L1-L2 顺手活 / 死代码删 / 命名修复** | **CHANGELOG 主记** |
| 跨项目反向学习吸收 | CHANGELOG 主记（+ PROP 如影响大）|

格式铁律：每条 4 行（改了啥 / 触发原因 / 验收结果 / 后续影响），≤ 10 行。

## 关键取舍

### 1. PS1 是权威实现，文档侧只解读

P4c / P4d 的逻辑同时写在 `agent/skills/项目体检-检查项-7-8.md`（维度 7+8）和 `agent/skills/状态推断.md`（推断 10）。如未来逻辑变化：

- 改 PS1 → 文档侧必同步（PROP 流程）
- 改文档不改 PS1 → 下次跑 PS1 暴露不一致

→ 工具是源头，文档是注解。

### 2. AGENTS.md 是项目根唯一例外的"AI 入口文件"

按 ADR-007 / AI 边界.md C 类，项目根的 `_framework` 级文件 AI 不能动。但 AGENTS.md 特殊 —— **它就是给 AI 看的入口**，由 AI 写最合适。

→ AGENTS.md 修改归类为 **B 类**（需 zlbdh review，不需 PROP/ADR 流程，类比 README.md）。

### 3. CHANGELOG 是单调增长文件 —— 进 RETRO 议题持续监控

CHANGELOG 设计上**只加不删**，会随项目演进**持续增长**。PROP-017 收档前 4453B，加 P3/P4/P5 三条 entry 后到 6821B（软建议区）。

不在本 ADR 解决，进 RETRO-008 议题"CHANGELOG 老条目归档策略"（按 Sprint / 半年归档冷条目到 `agent/CHANGELOG-2026H1.md`）。

### 4. PS1 Windows 强依赖 —— bash 镜像逻辑兜底

`pwsh -File` 需 PowerShell 7+，但 zlbdh 用 PS5.1（推荐 `powershell -File` 兜底）。Codex 在远端 Linux 跑，**不能跑 PS1**。

→ 推断 10 / 项目体检-检查项-7-8.md 提供 bash 镜像逻辑（同阈值 / 同维度），保跨工具一致性。

**bash 镜像执行时机**（PM 自审 #14 补）：
- **session 起手强制**：Codex / Cowork 起手时由 `agent/skills/状态推断.md` 推断 10 自动跑 P4d bash 镜像（状态.md 新鲜度）
- **重大改动后主动跑**：framework 重构 / agent 文件大改 / 引用链调整后，AI 主动跑 P4c bash 镜像（引用频率）兜底
- **PS1 实跑优先**：能跑 PS1 时（zlbdh 在 Windows）以 PS1 输出为准；bash 镜像仅当 PS1 不可用

## 后果

### 收益

- **新工具入场提速** —— 新 AI session 5 秒上手（读 AGENTS.md），不需扫 agent/ 整套。
- **健康度可观测** —— PS1 一键跑，结果客观（不依赖 AI 判断 / bash 缓存）。
- **小动作不丢痕** —— CHANGELOG 承接 L1-L2，不再走重 PROP 流程。
- **mount 缓存陷阱可绕** —— PS1 在 Windows host 直跑，不经过 mount，决策点 ground truth 可信。
- **跨项目双向学习成熟化** —— 后续{{PROJECT_NAME}}机制可反向输出到姐妹项目，形成双向迭代。

### 代价

- **三处冗余维护** —— AGENTS.md / PS1 / CHANGELOG / agent/skills/ 都讲"如何检 + 如何解读"，必须保持同步。本 ADR 锁定 PS1 为权威。
- **文件大小副作用** —— PROP-017 P5 后 `agent/skills/项目体检.md` 6177B（警戒）、`agent/CHANGELOG.md` 6821B（软建议）。属预期内，进 RETRO 议题。
- **PS1 跨平台兼容性** —— Codex 远端无 Windows shell 不能跑 PS1，靠 bash 镜像逻辑兜底。

### 风险与缓解

- **PS1 / 文档逻辑漂移**：未来若改逻辑只改一边，下次体检会暴露不一致。缓解：本 ADR 锁定权威；文档侧 PR 必勾 PS1 review。
- **CHANGELOG 失控膨胀**：5 年内可能涨到 50KB+。缓解：RETRO-008 议题"归档策略"提前规划。
- **AGENTS.md 失同步**：agent/ 大改但 AGENTS.md 未更新时，新工具看到的入口和实际不一致。缓解：AGENTS.md 已进 PS1 P4a 必查项（首行）。
- **PS1 修改本身受 mount 缓存影响**（PM 自审 #14 补）：PS1 是 ground truth，但**修改 PS1 时**仍是 Cowork Edit/Write，受 mount 同步陷阱（PM 自纠 #9/#12 实战 3 次踩坑）。缓解：PS1 重要改动后强制 Windows 实跑一次验证（已成 PROP-017 实战流程），bash stat 不可信。
