## 决定

### 决定 1：跨工具信源信任度规范（议题 D 升级）

跨工具操作 framework 文件或决策时，**信源优先级**：

```
PS1（Windows host 真值，权威）
  > Read 工具（host file 直读，绕 mount cache）
  > Cowork bash wc -c / stat / ls（mount 缓存可能 stale）
```

**Cowork PM 防御 4 规则**（PROP-018 P2 已实施到 PS1 P4e + agent/skills/项目体检-检查项-7-8.md 检查项 9）：

| # | 规则 | 触发场景 |
|---|---|---|
| ① | Read 复核警戒 / 危险文件大小 | 文件接近 6500B 警戒 / 8KB 软建议 |
| ② | Edit/Write 改 framework 文件后立即 Read 复核 tail | 验证实际写入生效 |
| ③ | git status 不一致时刷新 mount 或重启 Cowork session | bash 看不到新文件 / 看到旧 diff |
| ④ | P4b 警戒清单字节数：信 PS1 不信 Cowork bash | 双方数字不一致时 |

**bash 可信时机**：只读 grep / 跨文件统计 / 不关心字节精度 / 跨 session 已过几分钟。
**bash 不可信时机**：刚 Edit/Write 后立即 wc -c / 判断文件超警戒线决策依据。

### 决定 2：AI 配置默认值 C 类 → B 类下放 + 6 护栏（议题 V 升级，类比 PROP-012）

AI 配置默认值（baseUrl / model / apiKey）从 AI 边界.md **C 类（"永不动"）→ B 类（zlbdh review + 6 护栏）**。

**6 护栏**：
1. `.env.local` 必须在 `.gitignore`（4 层防护：`*.local` + `.env` + `.env.local` + `.env.*.local`）
2. key 不写代码 / 不写 git tracked 文件
3. **dev mode gate**：用 `VITE_DEV_DEFAULT_AI_APIKEY` 存在性判断（**注意：原 `import.meta.env.DEV` 在 debug APK = false，实战修订**）
4. user 配置覆盖优先级最高：IndexedDB > localStorage > dev-default > 无（fallback）
5. APK 不传第三方（zlbdh 自用 debug APK）
6. 不上 Google Play / 公开 release

### 决定 3：真敏感 / 半敏感泄漏规范（议题 X 升级）

**分级**：
- **真敏感**：API key / secret token / 账户密码 — **绝对不进 chat / 不进 git / 不进 conversation**
- **半敏感**：baseUrl / model 名 / 内部 PROP 编号 — chat 简版尽量用占位符（如 `<你的网关 baseUrl>`），不强制
- **公开**：FOSS lib 名 / 公开 doc 链接 / 等 — 无限制

**已实战 #1**（议题 X）：zlbdh 在 chat explicit override 贴 API key → PM 配 `.env.local`。**缓解动作**：Sprint-4 收尾后 rotate key（10 秒到中转网关后台）。

**未来 explicit override 模板**：
1. PM 警告（明示风险 + 替代方案）
2. zlbdh 明确 approve（"我承担风险，请处理"）
3. 行动后 PM 主动提醒 rotate / 撤销

### 决定 4：PRD 必含「空态 / 默认值 / 边界值」章节（议题 P 升级）

**元规则升级**：所有 PRD（Sprint-X 需求清单.md 内 F-XXX 段 + PROP）必含「空态 / 默认值 / 边界值」明示章节。

**示例（F-PREP-1 验收修订后的格式）**：
```
### 用户输入边界
- 数量字段留空 → 默认值（count 类: 5 / decimal 类: 0.2）
- 数字字符串 → 解析为对应数值
- 显式 0 → 用户故意 0（区分 "留空 → 默认" vs "显式 0 → 真 0"）
- null / undefined / "" → 三态拦截（议题 M 表单空值统一处理）
```

**落地**：升级到 `agent/rules/写PRD.md`（PROP-018 P6-2 收档时写）。

### 决定 5：四类角色分工规则（议题 AF 升级，PM 自纠 #38/#39 落地）

**角色分工四类**：

| 类别 | 谁做 | 范围 |
|---|---|---|
| **framework 内务** | **PM 自己做** | agent/ 内 .md + tools/ + 项目根 AGENTS.md / 状态.md / README.md + PROP / ADR / RETRO 编辑 + 议题 backlog 管理 |
| **业务代码** | Claude Code（PM 写交接卡）| {{APP_REPO_DIR}}/src/ 业务功能 + {{APP_REPO_DIR}}/android / {{APP_REPO_DIR}}/package.json |
| **测试代码** | Claude Code（PM 写交接卡）| {{APP_REPO_DIR}}/src/__tests__/*.test.js mock 设计 + describe + 用例编写 |
| **闭环验证** | Codex（Claude Code 写完后接力）| bump version + build APK + 跑 vitest 复核（确认 N/N + 0 regression）+ 真机 smoke + 截图 + git commit/push（ADR-016 B 类 6 条件）|

**反例**（PROP-018 P1/P2 我做错的）：
- PM 写交接卡让 Claude Code 改 `agent/CHANGELOG.md` ❌（应 PM 自己做）
- PM 写交接卡让 Claude Code 改 `tools/check-operating-system.ps1` ❌（应 PM 自己做）

**测试两层概念**（议题 AF.2，PM 自纠 #39 落地）：
- **写测试代码** = Claude Code（编写 mock + describe + 用例）
- **跑测试** = Claude Code 自测 + **Codex 闭环复核**（确认 N/N + 0 regression）+ **真机 smoke = Codex 专属**

**落地**：升级到 `agent/agents/AI边界.md` A 类 / B 类清单（PROP-018 P6-2 收档时写）。

## 关键取舍

### 1. 6 议题合并 1 份 ADR vs 拆 6 份

**选合并**：6 议题主题相近（元规则 + 跨工具护栏），紧凑 vs 档案膨胀。如果未来某议题大幅演进（如议题 D 升 PS1 自动 bash vs PowerShell 对比），再单独拆 ADR-023+。

### 2. 议题 AG（PS1 单测）不在 PROP-018 范围

议题 AG 是评估候选不是必修。Pester 引入需要 0.5-1d 工作量 + 测试设计成本，**PROP-018 已 4-5h scope**。留 RETRO-009 + PROP-019。

### 3. 议题 D MVP 静态文本而非动态对比

PS1 在 Windows host 跑无法直接读 Cowork bash 视角，跨工具自动同步逻辑复杂。**MVP 是认知 + 流程化防御**，未来 ADR-023 候选升级到「PS1 输出 baseline + Cowork 工具 diff 复核」机制（远期）。

### 4. ADR-022 标题取舍

选当前长标题「跨工具同步规范 + AI 配置 C 类下放 + 角色边界规则」—— 自描述，未来查找 grep 关键词命中率高。

## 后果

### 收益

- **跨工具信源信任度规范明示**：议题 D 10 次实战累积转化为流程规则，PM 不再凭直觉判断 bash vs Read vs PS1
- **AI 配置 C 类下放标准模式**：跟 PROP-012 git push 下放复用，6 护栏锁定 + 实战修订（DEV gate → APIKEY 存在 gate）
- **真敏感 / 半敏感泄漏分级**：议题 X explicit override 流程化 + 缓解动作（rotate key）规范化
- **PRD 元规则升级**：RETRO-007 卡 1 + F-PREP-1 #07 同模式两次踩坑后，PRD 模板强制空态/默认值章节
- **四类角色分工规则锁定**：PM 自纠 #38/#39 教训系统化，未来 PROP/feature 不再混淆角色边界
- **PS1 测试评估候选**：议题 AG 留 RETRO-009 + PROP-019，避免 PROP-018 scope 膨胀

### 代价

- **ADR-022 文档长度**：约 9-10KB（合并 6 议题导致超 ADR 平均 5KB，但 Docs/ 范围豁免）
- **agent/rules/写PRD.md 待更新**：PROP-018 P6-2 收档时同步加空态/默认值章节模板
- **agent/agents/AI边界.md 待更新**：A 类 / B 类清单加四类角色分工 + framework 内务明示
- **跨工具规则学习曲线**：新工具加入项目时（如未来 Cursor / Devin）要学习信源优先级

### 风险与缓解

- **议题 V 6 护栏 ③ DEV gate 实战修订**：debug APK 实际 `DEV=false`，已改为 `VITE_DEV_DEFAULT_AI_APIKEY` 存在性 gate（PROP-018 P2 已修）
- **议题 D 静态文本提醒可能被忽视**：PS1 P4e 每次跑都输出，跨 session 持续提醒（认知防御）；缓解：议题 D 升级到 ADR-023 候选动态对比（远期）
- **议题 X 实战缓解措施依赖人工 rotate**：未来如果发生 #2、#3 次泄漏，可能需要"半敏感"分级升级到"真敏感"（议题 X 阈值收紧）
- **议题 AF 四类分工边界模糊场景**：如 PM 改 {{APP_REPO_DIR}}/build-apk.bat（既是 framework 又是业务流水线）— 默认走 PM 自己做（framework 优先）
