# ADR-037 · 敏感刀对抗审查 + 隐私 lens

- **状态**：现行
- **日期**：2026-06-09
- **决策人**：zlbdh approve / 沉淀 PM 起草（PROP-043）
- **关联**：RETRO-017（armedClear 泄漏 + 过敏 5 路）+ RETRO-020（便当米饭蜡样芽孢杆菌 + M 偏离隐私）/ 元规则 DT

## 背景

数据类/安全类/不可逆刀多次在 ship 前对抗审查（ultracode）中抓到真问题：armedClear 二次确认绕过（v3.27）、过敏 5 路注入缺口（v3.29）、便当米饭蜡样芽孢杆菌「炒饭综合征」复热去不掉毒素（v3.39 真伤人隐患）、M 偏离记录敏感行为数据（v3.42 加每条可删除）。这些都不是普通功能 bug，是放出去会伤用户/泄隐私的红线。需把「哪些刀必审 + 数据刀必审隐私维度」固化。

## 决定

- **DT 敏感刀对抗审查铁律**：满足任一条件的刀，ship 前**强制 ultracode 对抗审查**（多透镜对抗验证，非自审）：
  - 触及用户数据删除 / 不可逆操作（armedClear 类）
  - 健康/食品安全建议（食物/复热/保鲜/运动负荷）
  - 数据 schema 迁移（叠加 ADR-036）
  - 注入 LLM prompt 的用户敏感数据
- **数据刀隐私 lens**（DT 的审查维度之一，敏感数据必查）：
  - 最小化：原始敏感内容截断（按 code-point 防 emoji 切坏）
  - 仅本地：敏感数据存本地 IndexedDB，聚合脱敏后才可进 prompt（原句/reason 永不进 prompt/网络）
  - 用户可控：敏感记录每条可删除
  - 不批判：呈现语气中立（「了解自己的节奏」非「惩罚/失败」）
- 纯确定性/纯复制/UI 展示类刀（无数据/安全/不可逆）豁免，免审查（保速度）。

## 后果

### 好处
- 伤用户/泄隐私的红线在 ship 前拦截（米饭毒素、过敏漏注入若放出去是真事故）。
- 隐私 lens 给数据刀一个固定的敏感数据处理清单，不靠临场想。

### 代价
- 敏感刀 ship 前多一轮对抗审查（时间成本）。
- 隐私 lens 4 条增加数据刀设计约束。

### 后续如果反悔了
- 去掉强制审查退回自审即可，但重新暴露红线放出风险。强烈不推荐（这是用户安全底线）。

---

## 备注
本 ADR 与三类行为铁律（C 类红线）衔接：C 类红线说「什么绝不能动」，DT 说「碰了相邻区域必审」。审查机制本身（ultracode 多透镜）属工具能力，本 ADR 只规定「何时必用」。
