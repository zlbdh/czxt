| 类别 | 技术 | 版本 | 说明 |
|---|---|---|---|
| 框架 | React | 18.3.1 | UI 库 |
| 构建 | Vite | 5.3.4 | 快、热更新好 |
| 路由 | React Router | 6.30.3 | HashRouter（Capacitor 友好） |
| 样式 | Tailwind CSS | 3.4.6 | 工具类 + 自定义 utilities |
| 图标 | lucide-react | 0.383.0 | SVG 图标库 |

| 图表 | Recharts | 3.8.1 | Pie + Line |

| 测试 | Vitest | 4.1.5 | 单元测试 |
| 转译 | esbuild | （内嵌 Vite）| 验证语法用 |

## 为什么这个组合

- **React 18** — 团队熟，生态好
- **Vite 而不是 Next.js** — 自用 SPA 不需要 SSR，构建快
- **Capacitor 而不是 React Native** — 同一套 web 代码可以装手机也可以做 PWA
- **Dexie 而不是直接 IndexedDB** — 写起来像 ORM，可读性高
- **Tailwind 而不是 CSS Modules** — 改样式不用切文件，原型快
- **HashRouter 而不是 BrowserRouter** — Capacitor 的 webview 加载本地文件，BrowserRouter 会跳路径找 404

## 依赖文件位置

- `package.json` — 顶层
- `package-lock.json` — 锁定版本
- `node_modules/` — 装出来的（`.gitignore` 排除）

## 构建产物

- `dist/` — Vite build 出来的静态资源
  - `dist/index.html`
  - `dist/assets/index-XXX.js`（主 bundle）
  - `dist/assets/charts-XXX.js`（recharts 拆包）
  - `dist/assets/storage-XXX.js`（dexie 拆包）
  - `dist/assets/react-XXX.js`（react 拆包）
  - `dist/assets/index-XXX.css`
- `android/app/build/outputs/apk/debug/app-debug.apk` — 最终 APK

## 加新依赖前问自己

1. 是否真的需要？能不能 30 行代码自己写？
2. 是否会让 bundle 大幅增长？
3. 是否有持续维护？最后一次发版多久前？
4. 是否会跟现有依赖冲突（多版本 React 等）？

如果有疑问，先不加。原则：**精简优先，少即是多**。
