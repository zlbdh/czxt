## 1. 5 层架构

```
┌─────────────────────────────────────────────────┐
│  🟦 索引层 (Index)        ← 导航 / 不存知识      │
│  README × 6 / INDEX.md                           │
├─────────────────────────────────────────────────┤
│  🟢 真相层 (Source of Truth)  ← 业务真相          │
│  Docs/1-需求文档/   (PRD / Sprint 清单 / 历史)   │
│  Docs/3-开发文档/adr/ (18 个 ADR 决策档案)       │
│  Docs/7-复盘/       (6 个 RETRO 复盘档案)        │
│  确认改动/已审批/已完成/ (14 个 PROP 提议档案)    │
│  {{APP_REPO_DIR}}/             (业务代码 + git 历史)         │
│  apk/              (APK 历史)                    │
├─────────────────────────────────────────────────┤
│  🟡 规则层 (Rules)        ← AI 行为规范           │
│  agent/agents/   (AI 边界 A/B/C)                 │
│  agent/workflows/ (实施循环 / 交接卡 / git 流程)  │
│  agent/skills/   (项目体检 / 状态推断 / 跑测试)   │
│  agent/rules/    (写代码 / 改动分级 / 技术约束)   │
│  agent/mcp/      (MCP 接入)                      │
├─────────────────────────────────────────────────┤
│  🟠 工作层 (Active Work)  ← 进行中               │
│  确认改动/待审批/ + 已审批/进行中/                │
│  交接区/待接手/  (当前等接手的卡)                │
├─────────────────────────────────────────────────┤
│  🔴 快照层 (Snapshot)     ← 跨 session "心电图"  │
│  状态.md  (顶部摘要 + 进度速查 + 推断决议)        │
│  交接区/已接手/ (历史档案) + 历史归档/(30 天前)   │
└─────────────────────────────────────────────────┘
```

## 3. 上下文加载机制

### Claude 200K tokens 窗口 ≈ 约 800KB 中文 / 4-5 倍英文

### 实际单次 session 加载（起手 5 步）

| 起手 | 文件 | 大小 |
|---|---|---|
| 1 | `agent/INDEX.md` | ~3KB |
| 2 | `状态.md` | ~3KB |
| 3 | 最新待接手交接卡 | 5-10KB |
| 4 | `agent/skills/状态推断*.md` (3 个) | ~15KB |
| 5 | 按需 `项目体检*.md` (3 个) | ~17KB |
| **小计** | | **~50KB** ≈ 7% 上下文 ✅ |

### 风险等级

| 场景 | 占用 | 等级 |
|---|---|---|
| 起手 5 步 | 7% | ✅ 安全 |
| 起手 + 全部 agent/ | 25% | 🟡 注意 |
| 起手 + 全部 ADR + RETRO + PROP | 28% | 🟡 注意 |
| 「读所有文档总结一下」 | 65% | 🔴 危险 |
| 起手 + 业务代码全读 | 85%+ | 🔴 危险 |

### 当前保护机制

1. ✅ 起手 5 步**强制按需加载**
2. ✅ 状态.md 顶部**摘要 + 链接**（PROP-011 精简后）
3. ✅ 交接卡**分目录归档**（待接手 / 已接手 / 历史归档 30 天）
4. ✅ PROP-013 8KB 软建议（单文件不爆炸）
5. ⚠️ **没有"全局加载上限"保护**（如 "读所有 ADR" 会触发）

## 4. 信息流转 4 通道

### 通道 1：跨 session 同工具

```
session N: 状态.md (顶部摘要) → session N+1 起手必读 → 1 分钟续上
```

### 通道 2：跨工具协作（核心）

```
工具 A 完成实施
    ↓
写完整交接卡 → 交接区/待接手/YYYY-MM-DD-...md (6 段)
    ↓
发 chat 简版 ①-⑥ 给 zlbdh (5-10 行，PROP-014 强制)
    ↓
zlbdh 切到工具 B
    ↓
工具 B 起手 5 步 → 读最新待接手卡 → 接手
```

### 通道 3：长期决策演进 (PROP → ADR → RETRO)

```
痛点信号 → 写 PROP → zlbdh 审批 → 实施 → mv 已完成 → 写 ADR 档案化
                                                          ↓
                                              每 3±1 L3+ → RETRO 复盘
                                                          ↓
                                                    发现新模式 → 新 PROP 循环
```

### 通道 4：跨文件 cross-reference（被动）

每个 .md 末尾有「## 关联」段，指向相关文件。AI 读到「关联」就知道还能去哪查。

## 5. AI 之间是否「记得」（上下文继承）

| 工具 | 跨 session 记忆 | 来源 |
|---|---|---|
| **Cowork**（PM）| ❌ 每个新 session 失忆 | 起手 5 步重建 |
| **Claude Code**（Dev）| ❌ 同上 | 同上 |
| **Codex**（QA + DevOps）| ❌ 同上 | 同上 |
| **zlbdh**（Owner）| ✅ 长期记忆 | 用户自己 |

→ **AI 之间没有"共享记忆"** — 信息全靠文件系统中转（状态.md + 交接区 + ADR 等档案）。

→ 这正是为什么 **PROP-011 交接区机制** + **PROP-014 chat 简版强制** 这么重要 — 它们是"AI 之间的硬通货"。

## 6. 强项 + 弱点

### ✅ 强项
1. 5 层架构清晰（知识 / 规则 / 工作 / 快照 / 索引）
2. 按需加载（起手 50KB，不超载）
3. 跨工具流通成熟（交接卡 6 段 + chat 简版 ①-⑥）
4. 长期演进闭环（PROP → ADR → RETRO 跑了 14/18/6 次）

### 🟡 弱点（未来 backlog）
1. 没有「主题视图」（如 git 相关规则散在 3 目录）
2. 历史档案累积（18 张交接卡，未来 100 张时）
3. 状态.md 顶部「已接手列表」会膨胀
4. 没有全局加载上限保护
5. 「记忆冷热分层」缺失（18 个 ADR 同等加载）

## 7. 跨项目复用建议

如果你给**新项目**搭类似 framework：

### 最小可行版（MVP，1-2 天）
- ✅ `状态.md` 跨 session 快照
- ✅ 「PRD / 决策 / 复盘」三类档案（哪怕没目录分层）
- ✅ 起手必读 1-2 步（status.md + 最新进展）

### 标准版（1-2 周）
- ✅ 加 `agent/` 目录 + AI 边界 A/B/C
- ✅ 加 PROP/ADR/RETRO 生命周期
- ✅ 加交接卡机制（如多工具协作）

### 完整版（持续演进）
- ✅ 项目体检 + 状态推断 自动化
- ✅ 流程优先三阶判定 + 交接卡强制
- ✅ 文件大小按工具分层
