---

## 核心设计

| 特征 | 说明 |
|---|---|
| **防过载** | 强制不超过 3 项「现在动手」+ 5-10 项「进 backlog」 |
| **元规则复用** | 用本项目 PROP-014「流程优先三阶判定」（🔴/🟡/🟢）作为严重度筛子 |
| **阶段性停手** | 5 步流程每步暂停，让 Owner 在每个节点有 control 权 |
| **精简 PROP 模板** | 不照搬本项目复杂版（14 PROP / 18 ADR 是 1 个月积累），给新项目最小模板 |
| **「自然生长」原则** | 让规范在业务需要时补，不"一次性安装"整套 framework |

## 提示词（完整版，直接复制给目标 AI）

````
你的角色是这个项目的 PM / 架构师。我刚才让你跑了一次「开发操作系统体检」，
你已经输出了体检报告（含「优势」/「关键缺失」/「改进路径」三段）。

现在我需要你**主导修正**这些缺失。按以下 5 步流程，不要省略，每步暂停等我看。

═══ 输入 ═══

读你刚才写的体检报告（在对话上文 或 体检报告文件）。

═══ Step 1：流程优先三阶重分类 ═══

体检报告里给的 P0/P1/P2 是按"重要程度"分的，但实际**动不动手**要看
"是否阻塞当前业务"。按下面三阶重新判定每个「关键缺失」：

| 严重度 | 表现 | 行为 |
|---|---|---|
| 🔴 阻塞 | 影响当前业务实施 / 测试 / 发布（不修无法继续）| **立刻动手修** |
| 🟡 非阻塞 bug | 警戒误报 / 文档不一致 / 路径错（不修能继续，但留隐患）| 进 backlog 下次优化 |
| 🟢 优化机会 | "换种方式可能更好"（不修也无碍）| 长期监控，可能不做 |

═══ Step 2：输出三档清单（≤ 3 项 + 5-10 项 + 长期）═══

```
🔴 现在动手（≤ 3 项 — 不要贪多）：
1. <缺失> — <为什么阻塞> — <最小可行修正> — <估时 S/M/L>
2. ...

🟡 进 backlog（5-10 项）：
- <缺失> — <长期影响> — <建议什么时候做>

🟢 长期监控（不立刻做）：
- <缺失> — <什么信号触发再做>
```

→ 暂停，让用户看 + 决定是否继续。

═══ Step 3：给「现在动手」第 1 项写最小可行改进方案 ═══

⚠️ **不要照搬参考项目（{{PROJECT_NAME}}）的复杂 PROP 模板** —
那个项目花了 1 个月才到 14 个 PROP / 18 个 ADR 的规模，
新项目应该从最小开始。

用下面**精简模板**：

```
# 改进提议：<一句话标题>

## 目标
<最终状态描述>

## 为什么现在做（阻塞证据 1-2 条）
- ...
- ...

## 具体步骤
1. ...
2. ...
3. ...

## 测试 / 验收
- [ ] 验证点 1
- [ ] 验证点 2

## 不在范围（避免范围爆炸）
- ❌ 不做 X
- ❌ 不动 Y

## 估时
S（< 1 小时）/ M（< 1 天）/ L（1-3 天）
```

→ 暂停，让用户看方案是否同意。

═══ Step 4：实施 + 记录 ═══

按 Step 3 改进方案实施。完成后：
- 测试通过验证点
- 在 README.md 或新建 `改进日志.md` 加一条记录（日期 + 改了啥 + 效果）
- 如该项目有跨工具协作 → 写简短交接卡（5-10 行）告诉下一棒
- 如该项目是单人单工具 → 不需要交接卡，"日志条目"即可

═══ Step 5：下一项 / 或停手 ═══

第 1 项完成后：
- 🔴 还有剩 → 回 Step 3 处理下一项
- 🔴 全清 → 停手 + 报告：「🔴 全清，剩 🟡 N 项 + 🟢 N 项已进 backlog，你决定要不要继续修 🟡」

═══ 防过载护栏（强制遵守）═══

- ❌ 不要一上来推 14 个 PROP
- ❌ 不要先建 `agent/{agents,workflows,skills,rules,mcp}/` 整套目录
- ❌ 不要先建 `确认改动/` `交接区/` `状态.md` 整套基础设施
- ❌ 不要先写 ADR README 索引表（先写第 1 个 ADR 再说）
- ✅ 让规范"自然生长" — 业务需要再补，不"一次性安装"
- ✅ 最小可行 — 1 个改进 / 1 次验证 / 1 条日志即可

═══ 特殊情况 ═══

- 如体检报告显示「关键缺失为 0」（项目很健康）→ 跳过 Step 3-5，告诉我「无需修正，进入正常开发节奏」
- 如体检报告全是 🟢（都不紧急）→ 同上
- 如项目还在早期（< 50 行代码 / < 10 个文件）→ 也告诉我「项目过早，体检意义不大，先把第一版业务跑通再说」

═══ 你现在 ═══

请：
1. 复读你的体检报告
2. 按 Step 1 三阶重新分类「关键缺失」
3. 按 Step 2 输出三档清单
4. **暂停**，等我说「Step 3 做第 1 项」再继续

不要一气呵成跑完 5 步 — 每步暂停让我有控制权。
````
