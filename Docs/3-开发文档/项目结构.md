## 关键约定

| 元素 | 位置 | 备注 |
|---|---|---|
| 项目代码 | `{{APP_REPO_DIR}}/` | npm install / dev / build / test 都在此跑 |
| 文档 | `Docs/`（在根） | 跟代码同级，引用代码用 `{{APP_REPO_DIR}}/...` 或带"以 {{APP_REPO_DIR}} 为基准"约定 |
| 操作系统 | `操作系统/` + `能力资产/` | 当前 PM/agent/workflow/rules/tools 真源 |
| PM 工作区 | `PM工作区/` | 9 PM 私人沉淀与速查表 |
| 待办 | `确认改动/待审批/` + `已审批/` | AI 提议先入此池 |
| 输出物 | `{{APP_REPO_DIR}}/apk/` | APK 归档 |
| CI | `{{APP_REPO_DIR}}/.github/workflows/build-apk.yml` | 当前 Git 仓库根目录是 `{{APP_REPO_DIR}}/`，workflow 必须放在这里 |

## 代码层架构（{{APP_REPO_DIR}}/src/）

```
请求 / 用户操作
  ↓
features/* （UI 层 + 各页面状态）
  ↓
useAppData() actions（应用层 / 数据流唯一入口）
  ↓
db.* (Dexie / IndexedDB)        + ai.* (callAI)
                                + ledger / health / date / image / devices helpers
```

**强约束**：
- 不在组件里直接 `db.xxx` 或 `await db.xxx`，必须经 `actions.xxx`
- 不直接 `setData(...)`，必须经 `actions.xxx` 同步 db + state

## 改了 schema 之后必须同步的文件

1. `{{APP_REPO_DIR}}/src/shared/database.js` — schema + helpers
2. `{{APP_REPO_DIR}}/src/app/useAppData.js` — initial state + actions
3. `Docs/3-开发文档/数据库schema.md` — 表结构文档
4. （L4）`Docs/3-开发文档/adr/ADR-XXX-...md` — 决策记录
