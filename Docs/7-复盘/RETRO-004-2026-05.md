## 共性洞察 — 7 个

### 洞察 1：规则有效性的最佳证明 = 自打脸
- PROP-007 立完「≥6500B 必拆」
- PROP-009 自己破了规则
- ADR-010 触发 → Claude Code 体检发现 → PROP-010 拆完
- **<24 小时内闭环**

→ 规则不是装饰。**触发条件 + 响应速度 = 规则真实有效性**。

### 洞察 2：跨工具协作的真实痛点不是「复制粘贴」
- zlbdh 点出：「问题是状态及信息的流转」
- 同步包 3 段太粗 → 接手者要靠推断
- 解决：结构化 6 段（文件变更 / 测试状态 / 待办 checklist / 警戒 / Q&A）

→ **粒度对了，跨工具协作才真有用**。

### 洞察 3：Claude Code vs Codex 工具行为不一致
- Claude Code 自动把截图读入 chat → 累积超 API 限制（>2000px / "Could not process image"）
- Codex 截图保存到文件不自动读回 → 稳定
- → 同样指令「保存到文件不贴 chat」，Claude Code 做不到（工具流程强制）

→ **跨工具差异需要在 DoD 端能力对照表里明示**。当前对照表只区分了「能不能跑」，没区分「跑出来的副作用」。

### 洞察 4：mount 缓存严重影响 bash 验证
- Edit 工具改完文件后 bash `wc -c` / `grep` 可能滞后几分钟
- 多次踩坑（v9 / v15 / v17 / v20 / v23）
- **必须用 Read 工具**验证关键状态

→ 项目体检 + 状态推断 skill「已知限制」段明示。

### 洞察 5：跨工具 PROP 接力是新模式
- PROP-008：Codex 写 + 拆代码部分 → Claude Code 收尾形式
- 工具间「接力赛跑」需要交接卡正式格式（恰好 PROP-009 提供了）
- 时间顺序：PROP-008 启动 → PROP-009 完成 → PROP-008 收尾用上了新格式

→ **改动密度高时机制演化加速**。

### 洞察 6：DoD 「跑两个对账 skill」 + 项目体检 + 状态推断 真的捕捉到了问题
- 不是抽象 checklist
- 实际抓出 PROP-009 引入的 7029B 越线
- 抓出建议 / 决议追踪
- 抓出大文件警戒区
- → **体检 / 推断 skill 设计有效**，不只是仪式

### 洞察 7：4 件 L3+ 节奏 = 框架成熟期标志
- RETRO-001 时大改还很多（{{APP_REPO_DIR}} 迁移 / rules→agent）
