### 决议

1. **结构性解除** — 拆分后每个子文件 < 5KB 单独管理，后续 schema 改只改对应小文件，不会再触及 16KB 临界

## 📚 PM 自纠 #54 案例研究

### 触发事件

PM handoff 卡（2026-05-20-1430-PROP-025-PRD-起稿-Cowork到ClaudeCode.md）⑤ Step 7+8 与 AC2 内部矛盾：

- Step 7 让 index.js 用无后缀 import（`export * from './schema'`）
- Step 8 让 rm database.js + 靠 Vite 自动 resolve `database/index.js`
- AC2 硬约束「caller 0 改动」
- **但 verify 后**：4 caller 实际用 `'../shared/database.js'`（带 .js 后缀）
- Vite/Vitest 带后缀 import **不会** strip 后缀做目录解析 → rm 会全部破坏 import

### 错误根因

PM 起 handoff 卡时**没 grep caller import 风格**就假设了 Vite 目录自动 resolve 行为。这是 PM 技术假设错误，**与议题 BR（AI 模型推测错向）+ BP（Live LLM 401 误判 P0）+ F-DIARY-1 UI 拍方案错位 同模式**：**不默认推测前必先核实**。

### 正确取舍（Claude Code 拍板，PM 接受）

保留 database.js 为 655B **薄壳** `export * from './database/index.js'`。

**优势 3 条**：
1. **0 caller 改动**（比 PM Step 8 更严格满足 AC2）
2. **议题 BW 真正解除**（最大子文件 4989B / 16KB = 11395B 余量）
3. **薄壳 655B** 后续就算 caller 增加也不会越线（PROP-025 后再扩 schema 安全）
