## 🎓 学到的事（3 教训）

### 教训 1 — 单测必须覆盖真机调用链
W-3 v1 单测覆盖 buildPersonaPrompt 函数本身但没覆盖真机会跑的路径（只在 deviation 调用）。v2 修复 + 加真机路径回归测试。**未来 AC 设计必须明示「这条测试覆盖的是哪个真机路径」**。

### 教训 2 — Framework 工作有"复杂度阈值"
zlbdh 3 次「停下来自检」实证 — 当议题制造速度 > 关闭速度时，PM 必须**主动进入维护模式**（冻结新 PROP / 新 RETRO / 新 memory），专注业务功能 ship。**Sprint-8 起 PM 默认维护模式**，仅在 zlbdh 明确触发或业务功能 ship 需要时才起新文档。

### 教训 3 — 议题 AJ 留痕机制需要外部 trigger
PM 嘴上说「切角色帽子」但实际没跑 decision-checkpoint + 没在状态.md 留痕 — 长 session 后机制衰减。PROP-027 推荐方案 A（chat 简版 ⑥ 段强制留痕检查）但本质问题是「PM 自我观察能力不足」需要主动制度化外部 trigger（如 chat 简版 ⑥ 必含「PM 切换轨迹本次加了 N 行」）。
