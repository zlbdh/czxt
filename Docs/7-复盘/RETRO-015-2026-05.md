## 四、关键洞察

### 1. ⭐ 大文件 mount 操作不可信（读 stale + 写 truncate）→ Read 工具是唯一真值通道
今天三连击：① #92 verify 工具靠 Read 而非真机整跑、躲过 bug ② #94 Edit 改 21KB 截断尾部 ③ 状态.md 84KB 被 mount 读成残缺/错版（python decode err / bash tail 停 L589 / grep 假阴性）。ADR-025 只覆盖「读 stale」，**写 truncate + Read-工具-唯一真值 是新角度** → 候选升 ADR-033。

### 2. verify 工具自己也要被 verify（#92）
「⏳ 真机 verify 留 zlbdh」= 验证债。加新检查段只测新段、不真机整跑 → 旧残留躲过。**改 framework 工具后必真机整跑**。

### 3. 源码 grep 契约 + 拆分重构，AC 应预先豁免「grep 目标跟随」（#95）
Chat.test.js 是源码 grep 契约，「0 测试改动」与「<8KB」数学互斥。Claude Code 逮住升 PM、选 A（grep 目标随代码搬家 / 断言不变）→ 议题 CD「PM 起卡 verify checklist」要加「grep 测试契约」维度。

### 4. ✅ 跨工具 9 PM 流程首次完整业务 ship 成功（正面）
Cowork 技术 PM 出卡 → Claude Code 开发 PM（逮 AC 矛盾、升级而非硬干）→ Cowork 项目 PM 拍板 → Codex 测试发布 PM 真机 ship → Cowork 收档。**议题 CD 防御机制实战有效 / 0 业务事故 / v4.0 架构业务 ship 验证通过**（之前只验过 framework 治理 ship）。

### 5. ADR-031 预测「Sprint-11 0 复发」未达（#93 执行层复发 1 次）
RETRO-014 预测 ADR-031 固化后 Sprint-11 0 复发。实际 #93 问句代替拍板 = 执行层复发 1 次（zlbdh「你是项目PM」trigger）。铁律本身没缺、是「收尾礼貌处」执行层破防 → 自检钩子：「要不要我…?」式收尾若答案显然=要 → 删问句直接做。
