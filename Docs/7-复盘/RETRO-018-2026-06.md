## 三、哪里卡（教训）

1. 🥇 **LLM 行为约束没定义可见 marker AC → smoke 无可验对象 → 3 轮往返**（本批最大成本）。控钠是「模型内心戏」，smoke 验不了隐性服从；第一刀卡上就该把「输出必须含什么字」定成 AC + smoke 判据同卡拍板。
2. **「替代」一词成两轮丢调料种子**——prompt 动词即 API：「酸味替代咸味」被模型执行成「把含钠调料从清单删掉」（过度服从）。诱导动作类动词（替代/删除/避免）必须配边界（不顶替原调料/只作补充）。
3. **maxTokens 4000 通用下限不够**：thinking 模型 + 注入块累积下是 path-dependent。采购路一直跑在 82-96% 悬崖边，r1-r3 每轮注入增量垫高 thinking，真机富档案先坠崖（简档过/富档挂）。
4. **prompt 注入空间 = 全路径共享预算**：每刀往同一 LLM 调用加注入块都在消耗所有场景的 thinking 头寸——注入块字数应全局记账。
5. **smoke 判据要预演「模型完美服从会输出什么」**：r2 采购 fail 实为模型把含钠调料全换成柠檬姜葱蒜（完美服从「替代」），判据反而落空。
6. 审查 harness 崩溃 ×1（lens 无结构化输出）：实施者未盲发、主动消除靶向矛盾——处置可接受，登记勿成惯例。
