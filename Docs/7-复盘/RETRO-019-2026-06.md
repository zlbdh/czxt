## 三、哪里卡（教训）

1. 🥇 **多层门控（RE 预筛→prompt 判定→normalize→兜底）改任一层必须全链同词验证**：G-F6「理发/洗牙」补了 RE 第 1 层（v3.36 r1）→ prompt 例表第 2 层没补 → v3.20「复诊」坑二进宫。每加一层锚词都要全链跟进。
2. **node-passes-device-fails tail 是反复出现的成本黑洞**：G-F6 node 真调 5/5 全过、设备屡挂（MiMo 端尾部形态差异 node 不复现）。终极解 = **确定性化**（regex authoritative 把 LLM 移出关键路径）+ DEV logcat 取证点（读设备实际值），而非继续 node 取证捉迷藏。
3. **PM 卡断言未实证（#54 同类，已自纠）**：G-F6 PRD 卡写「LLM 规则①⑥本就支持理发/洗牙」系勘察推断未真调，导致 r1 失败。勘察推断 ≠ 实证,涉 LLM 行为的卡断言应标「待 smoke 验」不写死。
4. **5 轮往返的根因是「LLM 输出不该进关键路径」**：canonical 周期句式本是机器可读的，第一刀就该正则为准（加固闸），而非求 LLM 给 recurrence。RETRO-018「行为约束定 marker」的姊妹教训：**结构化可机读的字段，正则优先于 LLM**。
