## 二、哪里顺（保留）

1. **hooks 自证价值（⭐ 当日最强信号）**：发布门禁一上线就揪出「假绿」（实为 npm.ps1 shim 坑），Stop hook（chat-summary）拦了它自己作者的回复缺 ①-⑦ 交接卡。新机制上线即抓真问题 = 设计对路的铁证。
2. **指针滞后根因定位精准**：子 agent 误报「当前 v3.6.5/Sprint-5」→ 根因=指针类活文档版本锚点滞后真知识源（{{APP_REPO_DIR}}/package.json）→ 全量同步 + 加 P4h 守卫一次性封堵，不只修当前值还修「会再漂移」。
3. **B-lite 受控并行 2 次试点 0 越界**：R1（26 文件 / 3 worker）+ R2（35 文件 / 33 worker + 2 单点）全部精确核（变更数 == 派工数）、禁并行哨兵未碰。AC2 派工卡互斥字段经 2 次实战验证有效。
4. **状态.md 切档字节守恒**：494KB→158KB（-68%）老轨迹归档 `状态-archive/`，字节级核对 0 丢失（ADR-033 大文件写截断教训内化为「切档先验字节守恒」）。
5. **负向测试入流程**：P4h 守卫上线即跑负向（README 改 Sprint-99 → exit 1 证响），发布门禁靠 ADR-033「改 framework 工具必真机整跑」抓出 npm.ps1 坑。守卫机制必须先证会响再信任。

## 三、哪里卡（坑 / 低效）

1. 🔴 **B-lite barrier 浪费（R1）**：worker 时长 spread **5.5×**（按文件数切 11/9/6 但 skills 更重，B=305s 拖死 A=79s/C=55s）。R2 改更均衡降到 **1.5×**。**教训：受控并行派工应按"预估工作量"均衡，不只文件数；或 pipeline 去 barrier** → 候选 **DU**（RETRO-021 已登记，本批 R1→R2 再加 1 个数据点）。
2. 🐛 **npm.ps1 shim 坑（险些误拦所有 push）**：PowerShell 里 `& npm <子命令>` 解析到 npm.ps1 会传错参（npm 收到 "pm"/Unknown command），发布门禁首次踩中，被 ADR-033 真机整跑抓出 → 用 `cmd /c "npm ..."` 解。**教训：framework 工具调 npm 一律走 `cmd /c`，绕过 PowerShell shim 解析。**
3. 🟡 **PM 轨迹时间戳手估系统性偏低**：本 session 手填轨迹时间比实测偏低数小时，被 `check-pm-tracking` 当场抓出。**教训：轨迹时间戳取 `Get-Date` 实测，不手填** → 候选 **DW**（RETRO-021 已登记）。
4. 🟡 **「Codex 有兜底、Claude 全靠自律」缺口**：之前只有 `.codex/hooks.json`，Claude 侧无 lifecycle 自动兜底（SessionStart/UserPromptSubmit/Stop 全靠自律）→ 本批补 `.claude/settings.json` 镜像。**教训：跨工具 framework 自动化要双工具对齐，不能一侧有兜底一侧裸奔。**
5. 🟡 **指针类活文档无锚点守卫先天会漂**：指针文档（README/AGENTS）天然滞后真知识源，P4g 列表一致性守不住「当前版本/Sprint」这种值锚点 → P4h 补盲区。**教训：凡「复制真知识源的值」的活文档，必有体检断言守住一致性，否则必漂。**

## 四、要不要改规范 / 升 ADR（action item）

**升 ADR 建议：PM 实体化 agent 调度模型 → 单升 ADR-038（本批主成果固化）。** 理由：

1. PM 实体化 agent 化是本日**核心架构决策**（zlbdh 4 次追问"为什么不开 agent"驱动），涉及 9 PM 全量 agent 化 + §四 重写 + 禁嵌套铁律——属 L4 架构级，应固化为永久 ADR；
2. 它是 **ADR-027（沉淀 PM 元-层 + 9 PM 四层）+ ADR-031（项目 PM 单点对外身份）一脉**的执行落地（把"角色模型"实例化为"agent 调度模型"），承上启下需 ADR 锚定；
3. PROP-044 已 approve、§三.5/§四已是现行规则、B-lite 经 2 次试点验证（RETRO-021+本批）——升 ADR 是把已生效机制锁宪法级，时机成熟。
