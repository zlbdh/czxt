## 二、哪里顺（保留）

1. **🥇 dogfood 实跑 > 纸面审计（本会话最强信号）**：多维审计派的 5 个 explorer 通读全框架都没抓到实例化递归套娃 BUG——只有**真把操作系统自装一遍**才炸出来（旧脚本递归复制含「正在生成的实例自身」的 `项目区`，无限套娃撑盘）。证据级教训：高危集成 BUG 靠读代码读不出，必须真机自举一遍。
2. **🛡️ 框架自带守卫反向 catch 作者（self-consistency 健康样本）**：P4a（copyItems 防回归）守住「项目区不得回到总递归清单」、PM 轨迹守卫（check-pm-tracking / P4f 口径）当场抓出手填时间戳偏差、readme-index 抓出 PROP 漏链 + 发布边界锚点缺口——三处守卫都主动拦住了项目 PM 自己这一轮的改动。守卫机制对「作者本人」有效，是它真在干活的铁证。
3. **修 BUG 顺手堵根因，不只修当前值**：套娃修复同步升 P4a 守卫（既防回归又防漏拷）；体检 scope 漏洞同步改 P4g 排除 `\本地实例\`——都把「会再犯」的口子一并焊死，不留「下次还得手修」尾巴。
4. **PROP-001 范围克制**：主动把噪声大的「单源文件 ask」（Plan B）和「git 命令级拦截」（Plan C / Phase 2）剔出本轮，只做历史归档 + 历史 APK 两类高价值低噪声路径，避免 scope 膨胀和 "rebase" 等词误报。范围收敛写进了 PROP 备选方案，决策可追溯。
5. **push 失败按铁律停手**：PROP-001 首推遇瞬时 Connection reset，按 B 类条件④停手不重试，zlbdh 授权后重试 1 次成功——没有在网络抖动上盲目重试。
6. **worker 执行质量达标**：worker 守住互斥写集（两份 hook + fixture）、codex 按自身契约走 additionalContext 软警告而非凭空发明 `permissionDecision`、并诚实报出主会话域的 readme-index 缺口（漏链/锚点），没有替主会话粉饰。

## 三、哪里卡（坑 / 低效 / 流程偏差）

1. **🔴 执行模型纠偏（本会话最该记的流程偏差，不美化）**：项目 PM 一开始把多刀 framework 写活全在主会话 inline 做（hat-switch 自己动手），被 zlbdh **两次纠正**才回到 ADR-038 调度模型——每个工作单元动手前先跑 Q7 判定、写活默认派 worker。直到 PROP-001 实现（轨迹 14:48）才是**本会话首次正确派 worker**。**教训**：ADR-038 刚升宪法级（RETRO-022），但「刚立的规则在下一会话第一时间就被作者自己违反」——说明 agent 调度还没形成肌肉记忆，Q7 应是每个写活单元的**前置闸**而非事后补。这正是「软规则失守」的活样本，呼应本轮 PROP-001 的同一动机（靠自觉不可靠）。
2. **🐛 PM 轨迹时间戳手猜偏差（复发，非首现）**：本会话又出现手写时间戳与真值偏差（手猜 12:50 / 真值 13:55，差约 66 分钟），被轨迹守卫抓出后改正。**注意**：这条在 RETRO-021/022 已两度登记为候选 **DW**（「轨迹时间戳取 `Get-Date` 实测，不手填」），本轮是**第三次复发**——候选迟迟没落地，问题就反复发生。**教训**：DW 已够格从「观察」推进到「机制」——脚本自动盖 `Get-Date` 而非靠人自律。
3. **🟡 体检 scope 对自托管实例排除不统一**：dogfood 把操作系统常驻 `项目区/本地实例/` 后，P4g frontmatter 扫描未排除该目录，自托管沙盒副本污染计数（35→71 翻倍）。本轮只修了 P4g 一处。**教训**：`项目区/本地实例/` 是「框架自托管自己」的合法常驻位，但**其它子检查（P4a 之外的 frontmatter/计数/残留/链接类）是否都已统一排除尚未排查**——只修一处是头痛医头。
4. **🟡 README 门面长期标黄到本轮才闭环**：frontmatter 34/35 这个缺口一直挂着（长期标黄），属「门面对外漏斗」缺口，本轮 dogfood（从外部视角看自己 README）才照出来。**教训**：模板自身的对外门面（README→AGENTS→00_总入口 漏斗）缺乏「以使用者视角定期自检」的触发器，靠日常体检的内部度量看不见。
5. **🧱 环境坑（已绕过，记账备查）**：commit 阶段 `Remove-Item` 被沙箱守卫误拦 → 改写仓库外 temp + `[System.IO.File]::Delete` 绕过完成 commit-msg。属环境/沙箱交互细则，无害但会复现。
