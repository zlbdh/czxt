# PM 自纠 #64 — PM 角色不应绑定特定工具 ⭐⭐⭐

## 一句话

PM 角色定义把工具名当角色名（「Cowork PM」「Claude Code PM」「Codex PM」）— 绑死了工具，违反"操作系统跨工具可移植"哲学。

## 触发场景

**时间**：2026-05-21 23:55 — 2026-05-22 00:00

**步骤**：
1. PM 起 task #95 终态愿景 v4.0 时写：「Cowork PM 内部 5 帽子」「开发 PM「Claude Code」」「测试发布 PM「Codex」」
2. zlbdh 4 处反复反问：「不要局限死 Cowork，Cowork 只是个工具，我们要能随便切换」「不要局限死 Claude Code」「不要局限死 Codex」「不要局限死工具，工具只是个载体」
3. PM 立即反思 — 把工具当 PM 角色名 = 锁死了工具可替换性

## 根因

- ❌ PM 写终态愿景时把"工具"和"角色"混为一谈
- ❌ 8 PM 矩阵 6 个用职责名（项目 PM / 操作系统 PM ...）但 2 个新增用工具名（Claude Code / Codex）— 不一致
- ❌ 设计哲学不一致：要么全部角色名 / 要么全部工具名 — 不能混着用
- ✅ 真相：**PM 角色 = 抽象协作角色；工具 = 可替换载体**
