## 六、教训

### 1. chat 简版 ⑦ 段硬约束只是"格式约束"
- 约束我**写了什么**（chat 输出）
- 不约束我**做了什么**（真写入）
- → 软规则 + 格式约束都需要**真验证机制**才能可信

### 2. Edit 工具返回 "updated successfully" 不可信
- Cowork mount 写入有 stale 风险（议题 BK 延伸 / 5/22 03:00 第一次截断）
- 我应该 Edit 后**强制 grep 真核**（不是关键字 grep，而是精确位置 + 行号）

### 3. 沉淀 PM 跨 PM 监督失守
- 沉淀 PM 在 task #102-#108 多次"框架体检"
- 没查"状态.md PM 切换轨迹是否在末尾段真有"
- 沉淀 PM 速查表必须加这条 verify

### 4. 5 次假报 = 议题 BC 工程一致性破坏
- 议题 BC 要求工程一致性 / Sprint 内 N 次同类操作必都成功
- 5 task 连续 Edit 失败 = 工程一致性严重破坏
