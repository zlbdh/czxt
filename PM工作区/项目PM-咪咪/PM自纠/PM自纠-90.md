## 错向

task #120 体检 README 体系时,只扫了项目根 README + 02_智能体 + PM工作区,**漏了**:
- `操作系统/00_总入口.md` —— 写"5 PM + 运营 + Dev/QA 占位" / 实际 v4.0 9 PM × 4 层

## 根因

task #120 体检 SOP 只查"README.md 文件",**没扩展到所有"入口类文件"**:
- `00_总入口.md`、`INDEX.md`、`AGENTS.md` 都是起手入口
- 但我体检脚本只 `find -name README.md`

→ **议题 DN 体检 SOP 升级**:不只 README,还要含 `00_*`、`INDEX*`、`AGENTS*` 等所有"入口类文件"。

## 修正

## 沉淀（议题 DN v2 SOP 升级）

体检入口类文件**必查清单**:

```
find . \( -name "README.md" -o -name "AGENTS.md" \
       -o -name "00_总入口*" -o -name "INDEX.md" \
       -o -name "总入口*" \) \
   -not -path "*/.git/*" -not -path "*/node_modules/*"
```

任一文件含以下关键词需核 vs v4.0 终态:
- "5 PM" / "6 PM" / "8 PM" → 应为 "9 PM × 4 层"
- "PM-产品经理" / "Dev-开发" / "QA-测试" → 旧 PM 命名（ADR-007 历史档案豁免除外）
- "v2.x" / "v3.x" 当前版本号 → 检查是否过期
