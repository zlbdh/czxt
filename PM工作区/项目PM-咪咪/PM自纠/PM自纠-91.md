## 错向

ADR README 表格停在 **ADR-025**（2026-05-19 收档时更新过）/ Sprint-10 我连起 ADR-029/030/031/032 4 个新 ADR + 之前 task #107 的 ADR-026/027/028 — **共 7 个新 ADR 全部没补进 ADR README 表格**。

frontmatter 还写"31 个 ADR" / 实际 32 个。

文件末"新增 ADR 后请更新这张表" —— 是软规则,**起 ADR 时我每次都漏**。

## 根因

### 1. 软规则失守同模式（同议题 BE / AJ）

"新增 ADR 后请更新这张表" = 软规则。起 ADR 时我满脑子在写 ADR 内容,**忘了同时更新 README 表格**。

任何**软规则 + 跨文件一致性**都易失守(议题 BE 起手必查就是同病 → ADR-029 升级)。

### 2. 议题 DN 体检 SOP 漏一个维度

ADR-032 决定 1-4 覆盖了:
- 入口类文件扩展定义
- 大文件可拆性 vs 核心性
- 历史档案豁免

**但漏了**:**列表类 README 内容一致性核查**(README 表格行数 vs 实际文件数)。

应该补 ADR-032 决定 6:**任何"索引/列表"类 README 必跟实际文件数核对**(如 ADR README 表格 vs `ls ADR-*.md | wc -l`)。

### 3. 维护责任未明确

README设计规范.md 写"操作系统 PM-框架管家"维护,但起 ADR 时我戴的是**操作系统 PM 帽子写 ADR** / 写完没切回再做"维护"步骤。

### 自动化建议（Sprint-11 候选）
集成到 `能力资产/tools/scripts/check-operating-system.ps1`:
```powershell
# P4? 列表一致性
$adr_files = (Get-ChildItem Docs/3-开发文档/adr/ADR-*.md).Count
$adr_readme_rows = (Get-Content Docs/3-开发文档/adr/README.md | Select-String "^| ADR-").Count
if ($adr_files -ne $adr_readme_rows) { Write-Warning "ADR 数量不一致: 文件 $adr_files vs README $adr_readme_rows" }
```
