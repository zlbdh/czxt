# 速查：ADR-022 决定 5 — 路径归属铁律

> 写 handoff / PROP / chat 前 30 秒核对 — 防 PM 自纠 #41/#42 同模式

## 判定铁律：看文件路径，不看任务性质

| 类别 | 文件路径 | 谁做 |
|---|---|---|
| **framework 内务** | `操作系统/` + `能力资产/` + 项目根（AGENTS.md / 状态.md / README.md）+ `Docs/` + `确认改动/` + `交接区/` | **项目 PM 切操作系统 PM** |
| **PM 私人工作区** | `PM工作区/<对应PM>/` | **对应 PM 自治；项目 PM 可调度，不跨界整理** |
| **业务代码** | `{{APP_REPO_DIR}}/src/` 所有 .js / .jsx（含 refactor + feature） | **开发 PM「实施者」** |
| **测试代码** | `{{APP_REPO_DIR}}/src/__tests__/*.test.js` | **开发 PM「实施者」编写；测试 PM「质量门户」定策略** |
| **闭环验证** | bump version + build APK + 跑 vitest + 真机 smoke + git commit/push | **测试发布 PM「闭环者」** |

## 9 PM 内部子角色（PROP-020 路径 D + v4.0）

```
项目PM「咪咪」(编排者) ─┬─→ 操作系统PM「框架管家」(操作系统/ + 能力资产/ + ...)
                       ├─→ 产品PM「需求拆解者」(Docs/1-需求文档/)
                       ├─→ 技术PM「修复决策者」(只读诊断)
                       ├─→ 测试PM「质量门户」(测试策略)
                       ├─→ 运营PM「运营咪咪」(PM工作区/运营PM-运营咪咪/)
                       ├─→ 开发PM「实施者」({{APP_REPO_DIR}}/src/)
                       ├─→ 测试发布PM「闭环者」(闭环验证)
                       └─→ 沉淀PM「沉淀者」(跨 PM 沉淀 / 元规则)
```

## 常见错向反例

- ❌ **PM 自纠 #41**：PM 没切角色帽子直接 Write `{{APP_REPO_DIR}}/src/anomalyDetector.js` → 应跑 decision-checkpoint → 切开发 PM「实施者」或写交接卡
- ❌ **PM 自纠 #42**：基于"性质是 framework 治理"判断错绕过路径白名单 → **铁律：看路径不看性质**
