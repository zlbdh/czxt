### 子 agent 调度速记（2026-06-14 / ADR-038）

- PM 是职责与权限，agent 是一次性执行实例。
- 除项目 PM 主会话外，每个 PM 的实际工作默认实例化为真实 agent：写=worker，只读诊断=explorer。
- 项目 PM「咪咪」拥有统一调度权 + 验收权；spawn 权不下放，其他 PM 可提出需要 agent，但不自治乱开。
- 单源禁并行文件仅由 agent 起草、主会话落最后一笔；发布动作单点不外包。
- 详见 [`01_架构/子agent调度机制.md`](01_架构/子agent调度机制.md)。

## 五、操作系统 vs 能力资产（PROP-029 v2）

```
操作系统/         (怎么协作)
├── 运行控制：角色 / 状态 / 交接 / 记忆 / 台账 / 工作流
└── 元规则池 + 38 ADR 永久现行

能力资产/         (用什么做)
├── rules/      执行硬规则（git 编码 / web-api 信源 / 改动分级 / 安全隐私）
├── shared/     业务能力（品牌词典 / 咪咪人设 / 跨分支协作机制）
├── mcp/        MCP 矩阵
├── tools/      构建脚本 + 依赖矩阵 + scripts/ 真可执行 PS1
├── skills/     工具能力 / 脚本（项目体检 / 状态推断 等）
├── agents/     执行型 agent（当前空 / 预留）
└── workflows/  执行型 workflow（当前空 / 预留）

项目配置/         (项目卡模板)
└── _模板.project.json

项目区/           (项目放哪里)
└── 本地实例/ 默认忽略真实项目内容
```
