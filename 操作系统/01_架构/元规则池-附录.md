# 元规则池附录

> 高频入口见 [`元规则池.md`](元规则池.md)。本附录承接解释性内容和演化历史。

## 一、与其他档案的层级关系

```
普通规则（日常 PROP / Sprint 决策 / handoff 卡）— 可改可废
   ↓ 累积 3+ 次同模式 + 评估
候选元规则（议题字母表 A-DE+）— 等评估
   ↓ Sprint RETRO 评估 + PROP 立稿
ADR（永久档案）
   ↓ 写入元规则池
✅ 永久化元规则（强制遵守 / 跨 Sprint / 跨 PM / 跨载体）
```

## 二、升级路径

```
PM 自纠（实战教训单次）
   ↓ 累积 3+ 次同模式（跨 Sprint）
候选元规则（议题字母表登记）
   ↓ Sprint RETRO 评估（沉淀 PM 主导起稿 / 项目 PM 拍板）
PROP 立稿 → zlbdh approve
   ↓
ADR（永久档案 / Docs/3-开发文档/adr/）
   ↓
写入元规则池 → ✅ 永久化
   ↓
跨 Sprint 强制遵守 + 跨 PM + 跨载体
```
