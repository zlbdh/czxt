## 工具切换时的迁移成本（2026-05-22 快照）

> 本段是候选路线推演，当前执行边界以主文和角色边界为准。

### 低成本切换（v4.0 已就位）

- ✅ Cowork → Claude Desktop / Claude.ai（主决策载体）— 0 迁移（所有 framework 在项目目录）
- ✅ Claude Code → Cursor Agent / Cline（实施载体）— 仅需新载体支持 `{{APP_REPO_DIR}}/src/` 编辑
- ✅ Codex → GitHub Actions / Jenkins（闭环载体）— 仅需配置 ADR-016 6 条件流程

### 中成本切换（待 v4.0 SDK 集成）

- 🟡 多形式自动化 Hooks（如 Codex+Claude 已有钩子 / Cursor 不同实现）
- 🟡 Skills SDK 集成（Anthropic Skills / 其他工具的等价物）

### 高成本切换（v5.0+ 探索）

- 🔴 Mem0 / Letta SDK 集成（记忆治理）
- 🔴 graph 记忆 entity-relationship
