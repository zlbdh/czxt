## 四、跨 PM 调度协议（PM 自纠 #66 / 不是"跨工具调度"）

PM 调度从来都是**跨 PM**（PM 角色之间）— 不论 PM 们用什么工具载体。

| 协议 | 内容 | 工具无关性 |
|---|---|---|
| **handoff 卡基础 ①-⑥ / framework 建议 ⑦** | 跨 PM 切换标准格式 | ✅ 纯 markdown / 任何工具可读写 |
| **chat 简版 ①-⑦** | 实施完成报告标准格式 | ✅ 纯 markdown |
| **状态.md 留痕** | PM 切换轨迹时间戳行 | ✅ 纯 markdown |
| **PROP / ADR / RETRO** | 决策档案系列 | ✅ 纯 markdown |
| **decision-checkpoint Q1-Q7** | 工作流判定（含 agent 实例化判定）| ✅ 工作流抽象 |
| **元规则池** | 当前永久 / 候选数以 `元规则池.md` 为准 | ✅ 工具无关 / 跨 PM 共享铁律 |

⭐ **关键设计原则**：跨 PM 调度的协议都是**纯 markdown / 工具无关** — 这样未来换工具（如 Cowork → Cursor）零迁移成本。
