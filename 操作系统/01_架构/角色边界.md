# Agent · AI 边界

> 先看路径，再看任务。路径决定 PM 角色、写入边界和是否需要交接。

本文件保留高频入口：9 PM 层级、白名单、敏感指针、调度入口；反例见附录。

## 快速判定

1. 先跑 [`decision-checkpoint`](../07_完整工作流/decision-checkpoint.md) Q1-Q7。
2. Q2 查本文件路径白名单：不在白名单就停，切角色或写交接卡。
3. 涉及敏感操作时同时查 [`三类行为铁律.md`](三类行为铁律.md)。
4. 写活按 Q7 判断是否派真实 agent；单源文件仍由主会话落最后一笔。

## 四类路径分工

| 类别 | 文件路径 | 责任 PM |
|---|---|---|
| framework 内务 | `操作系统/`、`能力资产/`、`确认改动/`、`交接区/`、`项目配置/`、`项目区/`、`Docs/3-开发文档/`、`Docs/7-复盘/`、根 `AGENTS.md`/`README.md`/`状态.md`/`实例化项目.ps1` | 操作系统 PM「框架管家」；项目 PM 验收 |
| 业务代码 | `{{APP_REPO_DIR}}/src/` 所有业务 `.js` / `.jsx` | 开发 PM「实施者」 |
| 测试代码 | `{{APP_REPO_DIR}}/src/**/__tests__/`、`*.test.js(x)` | 开发 PM「实施者」编写；测试 PM「质量门户」定策略 |
| 闭环验证 | bump version、build APK、vitest、真机 smoke、commit/push/tag | 测试发布 PM「闭环者」；ADR-016 B 类 6 条件 |

测试两层：写测试与开发期本地自测=开发 PM；发布/ship gate 的 vitest 复核、build、真机 smoke、发布复核=测试发布 PM。

## 9 PM 层级

```
主 PM（1）：项目 PM「咪咪」（唯一对外身份 / 编排者）
元-层 PM（1）：沉淀 PM「沉淀者」（跨 PM 沉淀 / 元规则池治理）
决策 PM（5）：操作系统 / 产品 / 技术 / 测试 / 运营
实施 PM（2）：开发 PM「实施者」 / 测试发布 PM「闭环者」
```

项目 PM「咪咪」是唯一对外身份；内部可切各 PM 帽子执行。运营 PM「运营咪咪」是 ADR-031 的平级 GTM 分支例外，按 `能力资产/shared/分支间协作机制.md` 执行。身份铁律见 [`角色边界-协作附录.md`](角色边界-协作附录.md)。

## 9 PM 完整路径白名单表

| PM 抽象角色 | 层级 | 路径白名单 | 不能做 |
|---|---|---|---|
| 项目 PM「咪咪」 | 主 | 全部 Read + 调度 8 PM + 收口写 `状态.md` / 交接卡 / chat ①-⑦ | 不绕过 Q1-Q7 直接动手 |
| 沉淀 PM「沉淀者」 | 元-层 | `PM工作区/沉淀PM-沉淀者/`、全部 Read；可触发检查脚本、起草元规则治理，最后落笔由项目 PM 主会话验收 | 不写业务代码；不动其他 PM 工作区；不直接改元规则池真源 |
| 操作系统 PM「框架管家」 | 决策 | `操作系统/`、`能力资产/`、`确认改动/`、`交接区/`、`项目配置/`、`项目区/`、`Docs/3-开发文档/`、`Docs/7-复盘/`、根 framework；`.gitignore` / `{{APP_REPO_DIR}}/.env.local` 的 baseUrl/model/apiKey 仅 B 类例外 | `{{APP_REPO_DIR}}/src/**`、APK、tag、push 禁区 |
| 产品 PM「需求拆解者」 | 决策 | `Docs/1-需求文档/`、`确认改动/待审批/` 写；`{{APP_REPO_DIR}}/src/`、`Docs/2` 只读 | 不写代码；不改 framework |
| 技术 PM「修复决策者」 | 决策 | 全部 Read / Grep / Glob | 不写文件；不改状态 |
| 测试 PM「质量门户」 | 决策 | 全部 Read / Grep / Glob | 不写 `*.test.*`；不跑 vitest/smoke |
| 运营 PM「运营咪咪」 | 决策 | 项目只读；`PM工作区/运营PM-运营咪咪/` 写；分支间待处理卡例外 | 不动 `{{APP_REPO_DIR}}/**`；不改 framework |
| 开发 PM「实施者」 | 实施 | `{{APP_REPO_DIR}}/src/` 业务代码 + 测试代码 | 不改 framework；不做发布闭环 |
| 测试发布 PM「闭环者」 | 实施 | version 元数据、`.gitattributes`、build、vitest 复核、APK、真机 smoke、commit/push、常规版本 tag/push tag（ADR-016） | 不改业务逻辑；不改 framework |

PM 私区规则见 [`PM工作区边界.md`](PM工作区边界.md)：只写自身 `PM工作区/<PM>/`；跨 PM 私区走 PROP 或该 PM 自决。

## 子 agent 调度边界

PM 是职责与权限，agent 是一次性执行实例。生成权和验收权在项目 PM「咪咪」，其他 PM 可提出需要 agent，但不自治乱开。

| 场景 | 默认 agent 口径 |
|---|---|
| 只读诊断、风险复核、历史查证 | explorer |
| 非单源 framework 写活 | worker 起草或写入，主会话验收 |
| 业务代码实现 | 开发 PM「实施者」worker，限 `{{APP_REPO_DIR}}/src/` |
| 测试策略 | 测试 PM「质量门户」explorer |
| 发布闭环、状态交接、commit/push/tag | 主会话单点收口 |

单源禁并行：`状态.md`、`交接区/`、`CHANGELOG.md`、`元规则池.md`、`角色边界.md`、`子agent调度机制.md`。agent 可起草，最后一笔由主会话落；完整机制见 [`子agent调度机制.md`](子agent调度机制.md)。

## 敏感操作速查

敏感操作先查 [`三类行为铁律.md`](三类行为铁律.md)。常规 commit/push/tag 为 B 类，归测试发布 PM 并满足 ADR-016；force/rebase/删分支/删改移 tag/release 为 C 类；真实密钥外传、tracked 密钥、用户数据删除、不可逆破坏均按 C 类停手。
