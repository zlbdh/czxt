# 长期产品化路线图

本文件记录 `czxt` 从本地操作系统模板走向长期产品化模板的阶段边界。它不替代 `README.md`，只做路线图和验收口径。

## 当前阶段

| 项 | 当前状态 | 证据 |
|---|---|---|
| 模板根 | 已独立为本地 git 仓库 | `git status --short --branch` 显示 `main` |
| 远端 | 已绑定 `https://github.com/zlbdh/czxt.git` | `git remote -v` |
| 项目区 | 已建立 | `项目区/README.md` + `项目区/清单.md` |
| 项目配置 | 已建立 | `项目配置/README.md` + `_模板.project.json` |
| 体检 | 模板根模式已通过 | `能力资产/tools/scripts/check-operating-system.ps1` |
| npm/package | 未进入 | 根目录不建立 `package.json` |
| 安装器/CLI | 未进入 | 当前仍用 `实例化项目.ps1` 显式参数 |

## 阶段定义

| 阶段 | 名称 | 进入条件 | 不做什么 |
|---|---|---|---|
| P0 | 本地模板整理 | 项目区、项目配置、模板根体检通过 | 不发布，不做 npm |
| P1 | `czxt` 首次入库 | 首个 commit 推到 `zlbdh/czxt`，远端 main 可拉取 | 不承诺安装器 |
| P2 | 多项目试装 | 至少 2 个项目卡片可通过 `实例化项目.ps1` 试装并体检 | 不把项目代码提交进模板 |
| P3 | 安装器雏形 | 项目卡片可驱动实例化脚本，支持 dry-run / apply | 不上 npm registry |
| P4 | package / CLI 评估 | 安装器稳定、版本策略明确、回滚路径明确 | 不为形式包装 package |

## P1 前验收

- `git remote -v` 指向 `https://github.com/zlbdh/czxt.git`。
- `check-operating-system.ps1` 无红项。
- `项目区/本地实例/` 被忽略，真实项目不会进模板仓库。
- `项目配置/` 只保留 `_模板.project.json`，不带具体项目卡。
- `README.md` 明确当前不是 npm package / 安装器 / CLI。
- 首次 commit message 使用中文，并说明“操作系统模板产品化起步”。

## 后续原则

- 先稳定模板结构，再考虑安装器。
- 先用 PowerShell 脚本跑通 Windows 本机，再评估跨平台包装。
- 项目实例永远通过显式项目卡登记，不靠隐式扫描猜项目。
- `czxt` 仓库只承载模板本体、规则、能力资产和脱敏样例；不承载具体项目卡、业务代码、密钥、APK 或用户数据。
