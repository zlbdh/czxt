## 反思 7 — 议题 AJ 留痕机制衰减防御（PM 自纠 #57 / #58）

**触发历史**：2026-05-20 zlbdh 2 次提示自检后，PM 才发现 09:30-15:30 约 6 小时 `状态.md` PM 轨迹 0 行。AJ 要求“切帽子前必跑 + 必留痕”，长 session 后衰减。

**真因**：
- ❌ 不是 decision-checkpoint 设计错。
- ✅ 是 PM 长 session 后没主动 trigger，忘跑 decision-checkpoint / 忘补 `状态.md`。
- ✅ PM 自我观察不足，需要外部 trigger 才发现。

**规则**：
1. 每次 PM 切角色帽子前：必跑 decision-checkpoint Q1-Q7；需要写 `状态.md` 时由项目 PM 主会话或对应白名单收口加 1 行，其他 PM/agent 只起草或报告
2. 每 5 个 chat 简版输出（约 15-30 分钟）必跑一次 mini 自检：「我本次切换了几次帽子？状态.md 加了几行？」如不一致 → 当场补登
3. 长 session 防衰减：chat 简版 ⑦ 必含“PM 轨迹加了 N 行（状态.md L<line>）”
4. zlbdh 任何“操作系统自检”提示：PM 必先停 + 自检留痕 + 再做事
5. 新对话起手：先 read `状态.md` 最新 PM 切换轨迹，了解上次 session 状态再开工

**PM 自纠 #57**：议题 AJ 留痕崩塌跨 Sprint 第 7 次同模式。

**PM 自纠 #58**：PM 自我观察衰减需外部 trigger；因此每 N 次 chat 自检，且 zlbdh 任何“自检”提示必停。

**议题 CK 候选监控**：状态.md 留痕机制衰减防御。

**议题 AJ v2 / PROP-027**：历史候选“自动 trigger 留痕”；现行仅 hooks/脚本提醒，不自动落笔。
