## 一、业界 SOTA 记忆架构 5 大方案（2026）

### 1. Mem0（最强 / 47K stars / 推荐）

**架构**：
- **3 tier scope**：user（用户级长期）/ session（会话级）/ agent（代理级）
- **混合存储**：vector + graph + key-value
- **自动 memory extraction**：从对话自动提取事实 / 关系 / 偏好
- **CRUD API**：外部记忆服务（agent 调用）
- **47K+ GitHub stars**（最大社区）

**特征**：
- 不是 agent 自己 reason memory（与 Letta 相反）
- 适合大多数团队 / 通用最强

### 2. Letta（前 MemGPT / Self-managing）

**架构**：
- Agent **自己在 loop 内** reason about memory
- 长期记忆 paging（短期 context + 长期持久层）
- 内置记忆 hygiene 机制

**与 Mem0 对比**：
- Mem0 = memory as service（外部 CRUD）
- Letta = memory in agent（agent 自治）
- **设计哲学相反**

### 3. LangMem（LangChain / LangGraph 原生）

**架构**：
- LangChain 官方记忆 SDK
- 支持 **3 类记忆**（episodic / semantic / procedural）
- 集成 fact extraction + behavior-level memory
- LangGraph 原生支持

### 4. Provider-managed（ChatGPT Memory / Claude Projects）

**架构**：
- 厂商内置记忆服务
- 用户无需自建
- 简单但锁定厂商

### 5. Custom in-app schema（DIY）

**架构**：
- 项目自建记忆 schema
- 完全控制 / 但维护成本高
- **当前{{PROJECT_NAME}}属此类**

## 二、记忆 3 类分类法（业界共识 / 借鉴认知科学）

业界 2025-2026 收敛到 **3 tier taxonomy**（基于认知科学 50 年研究）：

### Episodic（情景记忆）
- **什么**：历史事件 / 用户交互记录 / 时间戳的"做过什么"
- **业界例**：ChatGPT 对话历史 / Mem0 session memory
- **当前操作系统**：状态.md + 状态-archive/ + 版本时间线 + PM 自纠 artifact + 交接区/

### Semantic（语义记忆）
- **什么**：事实 / 概念 / 规则 / 不变的"知道什么"
- **业界例**：Mem0 user memory / LangMem fact extraction
- **当前操作系统**：05_记忆/INDEX.md + 议题全景.md + ADR / RETRO + 能力资产/rules + shared/

### Procedural（程序记忆）
- **什么**：how-to / 工作流 / SOP / "怎么做"
- **业界例**：LangMem behavior memory
- **当前操作系统**：6 PM 速查表（SKILL.md）+ 共享技能/ + decision-checkpoint workflow

## 四、当前优势 vs 业界差距

### ✅ 当前优势（业界对标已对齐 5 项）

1. **记忆 3 类完整覆盖** — Episodic / Semantic / Procedural 都有实例
2. **Scope 分层更精细** — 4 tier（Global / Project / Agent / Session）vs Mem0 3 tier（user/session/agent）
3. **跨 session 持久化** — 已落地（5/20 C 强化版从 AppData 迁项目目录）
4. **PM 自纠 artifact 机制** — 业界独有 / 类似 LangMem reflection
5. **跨 PM 边界明确** — PM 自纠 #59 跨界禁令 / 业界无对等

### 🔴 当前差距（业界 SOTA 3 项）

1. **维护方式**：100% 手动 vs Mem0/Letta 自动 CRUD
2. **检索方式**：纯 grep vs vector + graph + key-value 混合
3. **自动 extraction**：PM 主动写 vs 业界自动从对话提取

## 五、5 大改进建议（按 ROI 排序）

### 🔴 P0 改进 1：自动 memory extraction（自愈 PM 主导）

**当前**：PM 自纠 / 议题 / RETRO 全手动写

**升级**：
- 自愈 PM「反思者」（PM 自纠 #65 新增）主动扫描 session 提取候选记忆
- 自动生成 PM 自纠 artifact 草稿（PM review approve 后落地）
- 自动归类 3 类（episodic / semantic / procedural）

**收益**：议题 AJ 留痕崩塌真正根治（机制级解决）

**难度**：M / 自愈 PM 实现后

### 🔴 P0 改进 2：vector search 接入（mem0 集成）

**当前**：找 PM 自纠 / 议题 / 速查表全靠 grep + Read

**升级**：
- 给 `操作系统/05_记忆/INDEX.md` + PM 自纠 artifact + 议题全景 加 embedding
- 引入 mem0 或 LangMem SDK
- AI 按 description 匹配（取代 PM 手动 Q4 trigger 匹配）

**收益**：Progressive Context Loading 真 10x 节省（PROP-031 完整版）

**难度**：H / 需 SDK 集成 + 议题 CO 完整版

### 🟡 P1 改进 3：graph 记忆 entity-relationship

**当前**：议题之间关系 / PM 自纠之间关系隐式 / 元规则升级路径手动

**升级**：
- 显式建模 entity-relationship（议题 ←→ PM 自纠 ←→ ADR ←→ 元规则池）
- graph 数据库（Neo4j / 或轻量 .json schema）
- AI 推理"哪个 PM 自纠链接到 ADR-N？"等

**收益**：跨 Sprint 同模式自动识别（自愈 PM 升级）

**难度**：H / 议题 CV 候选

### 🟡 P1 改进 4：多形式自动化 trigger（更宽 Hooks）

**当前**：PowerShell check-pm-tracking.ps1 半自动

**升级**（zlbdh 提示 "Hooks 只是一种形式"）：
- file watch（fs events / watchman）
- cron schedule（定时检查）
- MCP tool events（mcp__scheduled-tasks）
- Codex/Claude 已有 钩子（不同工具不同形式）
- Chat output 事件

**收益**：议题 AJ 留痕真自动化（Layer 4 完整版）

**难度**：M / Sprint-9 工具能力到位

### 🟢 P2 改进 5：记忆 scope 4 tier 显式化

**当前**：4 tier（Global / Project / Agent / Session）已实质存在 / 未显式标注

**升级**：
- 给每个记忆文件加 YAML frontmatter `scope: global/project/agent/session`
- INDEX 中按 scope 分类
- decision-checkpoint Q4 速查表匹配按 scope 过滤

**收益**：记忆查找更精准

**难度**：L / Sprint-8 可立即做

## 七、记忆治理铁律（建议永久化）

基于业界 SOTA + 当前实战提炼：

### 铁律 1：3 类 + 4 tier 分类法（议题 CW 候选元规则）

每个记忆文件必标 type（episodic/semantic/procedural）+ scope（global/project/agent/session）

### 铁律 2：单一信息源（议题 CG 已永久化 / PROP-026）

每类记忆只在一个位置维护 / 其他位置仅指针

### 铁律 3：跨 PM 共享 vs PM 私人边界（PM 自纠 #59 已永久化）

每个 PM 自治自己记忆 / 不得越界 / 共享技能库独立维护

### 铁律 4：自动 extraction + 手动 review（议题 CX 候选）

记忆 candidate 自动提取（自愈 PM）/ PM review approve 后才入库 / 防 AI 幻觉污染
