# 工作流：发布流程

每个 Sprint / feature 结尾出一个 dev APK 给 zlbdh 装机。需要正式分发时走 release 闭环；GitHub Release / APK 分发由 zlbdh 手动裁决。

## Sprint / feature 结尾发布（dev APK）

### 1. 全量 QA
```powershell
cd {{PROJECT_ROOT}}\{{APP_REPO_DIR}}
npm test
npm run build
```
0 fail / 0 error 才放行。

### 2. 更新需求历史 / release notes
`Docs/1-需求文档/需求历史.md` 加新版本段。

### 3. 出 dev APK

优先使用 PowerShell 脚本，它会归档到 `{{APP_REPO_DIR}}/apk/`：

```powershell
cd {{PROJECT_ROOT}}\{{APP_REPO_DIR}}
powershell -NoProfile -ExecutionPolicy Bypass -File .\build-apk.ps1
```

如用 `build-apk.bat`，需确认是否只是 Gradle assemble；实际归档以 `build-apk.ps1` 或手动复制结果为准。

### 4. APK 归档路径

当前 dev APK 统一放在：

```text
{{APP_REPO_DIR}}/apk/{{PROJECT_NAME}}-vX.Y.Z-debug.apk
```

归档前确认目标 APK 不存在；已存在即停手，不覆盖历史 APK。若确需复用版本号，先按 B 类 version 规则重判。

### 5. smoke test

测试发布 PM「闭环者」（playbook 见 [`../02_智能体/测试发布PM-闭环者.md`](../02_智能体/测试发布PM-闭环者.md)，当前常由 Codex 承载）/ zlbdh 装机跑核心路径 smoke。根据反馈：
- 没问题 → 按 [`git流程.md`](git流程.md) B 类 6 条件由测试发布 PM「闭环者」push，更新 `状态.md` / 交接卡 / chat ①-⑦，再进入下一个 Sprint
- 有 bug → 修 → 重出 APK

## 正式 release 发布（vX.Y.Z）

### 1. 完成当前 release scope
确认对应 `Docs/1-需求文档/` 需求、需求历史、P0/P1 验收项都已闭环。

### 2. 出 release keystore（一次性）
此步由 zlbdh 手动执行；AI 不处理密码、不备份 / 外传 keystore、不把密钥材料写入 tracked 文件。

```cmd
keytool -genkey -v -keystore zlbdh-release.keystore ^
  -alias mimi -keyalg RSA -keysize 2048 -validity 10000
```
**记住密码！备份多份！**

### 3. 配 gradle release 签名
（看 `Docs/5-运维文档/APK打包指南.md` 的「打 release 签名 APK」段）

### 4. 出 release APK
```cmd
cd android
gradlew.bat assembleRelease
```
产物：`android/app/build/outputs/apk/release/app-release.apk`
拷到：`{{APP_REPO_DIR}}/apk/{{PROJECT_NAME}}-vX.Y.Z-release.apk`

目标 release APK 已存在则停手，不覆盖历史 APK。

### 5. 创建 git tag

仅测试发布 PM「闭环者」在 ADR-016 6 条件、`package.json` / Android 版本、APK 命名、release notes、交接卡版本全部一致后执行；其他 PM 不直接打 tag。

```cmd
git tag -a vX.Y.Z -m "vX.Y.Z release"
git push origin vX.Y.Z
```

### 6. 写 release notes
`Docs/1-需求文档/需求历史.md` vX.Y.Z 段写完整。
GitHub Release / APK 分发是 C 类，AI 不做；如需要由 zlbdh 手动发布。

### 7. 装机
- zlbdh 自己手机
- 想给亲友传 APK

## 紧急修复（main 直修）

### 触发条件
- 当前 release 装机后发现 P0 bug（数据丢 / 启动崩 / AI 不工作）

### 流程
1. 仍留在 `main` 分支，先确认满足 ADR-016 commit/push 6 条件
2. 修代码 + 加测试覆盖
3. 跑 QA 3 层
4. 出下一 patch debug APK 给 zlbdh 装
5. 验证后出 release
6. tag 下一个 patch 版本；不另开分支，因此没有分支合并步骤

### 发布间隔
- 小修 → 下一个 patch 版本
- 加新功能 → 下一个 minor 版本
- 大版本 → v4.0
