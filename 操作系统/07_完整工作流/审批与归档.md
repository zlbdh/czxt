# 工作流：审批与归档

PROP（提议）从「待审批」走到最终归档的全部状态转换规则。
项目 PM 收到 zlbdh 的审批信号后先跑 Q1-Q7，再按路径路由给产品 PM / 操作系统 PM / 测试发布 PM；工具只作执行载体，**不要凭记性**。

---

## 触发信号 → AI 必做动作

### A. 审批通过

**zlbdh 触发词**：「同意」/「批了」/「通过」/「OK 开干」/「可以做」/「✓」

**项目 PM 编排必做（按顺序）**：

1. **改 PROP 头部状态字段**：
   ```
   - **状态**：已审批 · 未收口 / 待排期（YYYY-MM-DD 起，对应 ADR-XXX 待写）
   ```
   日期填今天；ADR 编号查询见 §C。具体子状态可随进展改为「待重审 / 外部能力跟踪 / 待重估」，但主状态必须保持「已审批 · 未收口」。

2. **mv 文件**：
   ```
   确认改动/待审批/PROP-XXX-...md
       → 确认改动/已审批/进行中/PROP-XXX-...md
   ```

3. **更新 `确认改动/README.md` 计数**：
   - 待审批 -1
   - 进行中 +1
   - 「进行中」清单加一条

4. **【L3/L4 必做】路由产品 PM 建对应 PRD 条目**：
   在 `Docs/1-需求文档/Sprint-N需求清单.md` 加 F-XXX 条目（编号查询见 §C）。
   不建 → 开发 PM「实施者」实施时找不到 F 编号。

5. **【L4 必做】路由操作系统 PM 建 ADR 占位文件**：
   在 `Docs/3-开发文档/adr/ADR-XXX-...md` 建空文件（防止后面再撞编号）。
   头部至少写：`# ADR-XXX · <标题>` + `状态：起草中`。

---

### B. 拒绝

**zlbdh 触发词**：「不做」/「拒了」/「pass」/「先放着」（明确推了就是拒绝）/「✗」

**项目 PM 编排必做**：

1. **改 PROP 头部状态字段**：
   ```
   - **状态**：拒绝（YYYY-MM-DD，原因：……）
   ```
   原因填 zlbdh 给的理由（如果只是「不做」没解释，问一句再写）。

2. **mv 文件**：
   ```
   确认改动/待审批/PROP-XXX-...md
       → 确认改动/拒绝/PROP-XXX-...md
   ```

3. **更新 README.md 计数**：待审批 -1 / 拒绝 +1。

⚠️ **不要 `rm`**。拒绝的提议留档，半年后想起来还能查为什么 pass 了。

---

### C. 编号查询（写新 PROP / ADR / RETRO 前必跑）

**不要凭记忆猜编号**——这次 PROP-002 / ADR-005 都撞过。

```powershell
# 查 PROP 当前最大编号
Get-ChildItem -Path '确认改动' -Recurse -Filter 'PROP-*.md' -File |
  Select-Object -ExpandProperty BaseName |
  Sort-Object -Unique

# 查 ADR 当前最大编号
Get-ChildItem -Path 'Docs/3-开发文档/adr' -Filter 'ADR-*.md' -File |
  Select-Object -ExpandProperty BaseName |
  Sort-Object

# 查 RETRO 当前最大编号
Get-ChildItem -Path 'Docs/7-复盘' -Filter 'RETRO-*.md' -File |
  Select-Object -ExpandProperty BaseName |
  Sort-Object
```

新编号 = 上面输出里最大编号 + 1。

---

### D. 实施完成（已写在 [实施循环-DoD.md](实施循环-DoD.md)，重申引用）

DoD 里「PROP 收档」段：状态字段 → 已完成；mv 进行中/ → 已完成/；更新 README 计数。

---

### E. 中途弃用（罕见但有规范）

**触发**：实施到一半 zlbdh 说「这个不要了 / 方案错了，回滚」。

**项目 PM 编排必做**：

1. **改 PROP 头部状态字段**：
   ```
   - **状态**：已审批 · 已弃用（YYYY-MM-DD 弃用，原因：……）
   ```

2. **mv 文件**：
   ```
   确认改动/已审批/进行中/PROP-XXX-...md
       → 确认改动/已审批/已弃用/PROP-XXX-...md
   ```

3. **更新 README.md 计数**：进行中 -1 / 已弃用 +1。

4. **代码层**：如果代码已经改了一部分，发起一次回滚 PROP（PROP-YYY），走 §A 流程批准后再改代码。

5. **PRD 同步**：把 Sprint 清单里对应的 F-XXX 标「已弃用（原因：……）」，不删条目。

## 速查表

| zlbdh 说什么 | 文件去向 | 状态字段写什么 |
|---|---|---|
| 「同意」 / 「OK 开干」 | 待审批 → 已审批/进行中 | 已审批 · 未收口 / 待排期（日期 + ADR-XXX 待写） |
| 「不做」 / 「拒了」 | 待审批 → 拒绝 | 拒绝（日期 + 原因） |
| DoD 全过 | 进行中 → 已完成 | 已审批 · 已完成（日期 + ADR-XXX） |
| 「方案错了，回滚」 | 进行中 → 已弃用 | 已审批 · 已弃用（日期 + 原因） |

---

## 一句话原则

- **状态字段 + 路径，永远一致**。改一个必改另一个。
- **不删 PROP**。拒绝、弃用都留档。
- **写新编号前先 ls**。
