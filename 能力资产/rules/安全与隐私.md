# 规则：安全与隐私

隐私 / 凭据 / 数据保护的当前有效规则。C 类边界以 [`操作系统/01_架构/三类行为铁律.md`](../../操作系统/01_架构/三类行为铁律.md) 为最高门禁；本文件负责把敏感域的默认动作说清楚。

## 默认动作

- API key / token：不写入 git tracked 文件，不贴到交接卡，不发到外部服务。baseUrl / model 默认值若为非密配置，可在代码中按 ADR-022 护栏维护；涉及 `.env.local` 只按三类行为铁律 B 类护栏处理。
- 用户数据：IndexedDB、备份 JSON、真机数据默认只读都要谨慎；删除、外发、上传、迁移都属于 C 类，必须停手等 zlbdh 明确授权。
- 备份导出：JSON 可能含 chatMessages、tasks、health/profile 等信息；生成或处理前先说明风险，输出路径和内容范围必须可追。
- logcat / 截图：分享前先检查是否包含 key、手机号、账号、私人聊天、健康数据、路径中的个人信息。
- 云同步 / 第三方接入：默认不接。首次接入云端 sync、外部分析、应用商店发布隐私政策时，先起 PROP/ADR。

## 当前默认（基础规范）

- API key 不进 git。
- 自用 APP 默认不上云，数据全在本地。
- 备份导出 JSON 由用户自己保管。
- AI 不主动读取、保存、外发任何用户数据；用户显式给出的片段也只用于当前任务。

如出现更高风险场景，按下方触发条件升级为专项规范或 PROP/ADR。

## 何时升级（触发条件）

满足任一条 → 立刻起 PROP，把本文件升级为更完整规范：

- 🚨 **API key 即将进 git**（任何文件含真实密钥、token-plan-* 私密 token、Anthropic key 等被 commit）
- 🚨 **AI 第一次想读 / 改 / 备份 用户的 IndexedDB 数据**
- 🚨 **第一次 zlbdh 把备份 JSON 给别人 / 第三方**
- 🚨 **第一次想接入云端 sync**（违反「自用 / 不上云」初心）
- ✅ 准备发布到 Google Play / 应用商店（必须有隐私政策）

任何上面 5 条触发 → 立刻升级专项规则，不能拖。

## 误删恢复流程（PROP-007 P1-4）

AI 不应该删文件（C 类边界），但万一发生：

### 立刻动作（不要 panic）
1. **停手** — 不要再做任何 Write / Edit，避免覆盖更多
2. **告诉 zlbdh** — 清楚说明「我误删了 X 文件，原因是 Y，最后看到的内容长度约 Z 字节」
3. **不要重新写** — 凭记忆重写容易写错版本，可能比丢失更糟

### zlbdh 的恢复动作（按优先级）
1. **如果文件已 git commit 过** → `git restore <file>` （Windows 端跑）
2. **如果文件没 commit 但还在 sandbox** → 用 Read 工具看是否还能读到（mount 缓存可能还有副本）
3. **如果是 状态.md / PROP / ADR** → 可以从历史对话/截图恢复
4. **如果是代码** → AI 可以从内存中重写**但必须**给 zlbdh 看 diff 确认

### 防御
- AI **永远不主动**删文件（用 Edit 替换内容而不是删）
- 删文件前必须**向 zlbdh 明确确认**，按当前载体可用确认工具执行
- 删除授权只对本次、明确、非 C 类目标有效；不得把一次授权泛化为长期授权，也不得用它绕过三类行为铁律。
