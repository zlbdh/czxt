# 已知技术约束附录

高频速查见 [`已知技术约束.md`](已知技术约束.md)。

## 1. Mount 写入截断风险

历史现象：Claude 的 `Write` / `Edit` 工具对 mount 文件系统曾出现约 9105 字节硬截断；工具报告“成功”，但尾部不完整。2026-05-08 进一步确认：接近 8KB 的文件做小改动也可能触发整文件尾部消失。

历史实例：

- `Sprint-1需求清单.md` 约 9KB，Edit 后只剩 8250 字节，末尾损坏。
- `需求历史.md` 多次 Edit 后丢失 v1.x 整段与决策记录段。

历史建议（已被现行 P4b 分层细化）：

- 当时粗分：<6KB 可直接改；6-9KB 优先脚本写入或拆分；>9KB 优先拆分。
- 现行真口径：以 [`已知技术约束.md`](已知技术约束.md) 主文 + `check-operating-system.ps1` P4b 为准，按 6000 / 6500 / 8000B 分层。
- 关键判断：信 Windows host PS1 + Read 复核；bash `wc -c` 只作候选扫描。

历史验证样例（旧环境）：

```bash
wc -c target_file
tail -c 200 target_file | xxd | tail -3
python3 -c "open('target_file','rb').read().decode('utf-8', errors='strict'); print('OK')"
```

## 2-4. 沙箱 / 依赖 / 临时目录

- 沙箱不能直接出 APK：`dl.google.com`、`maven.google.com`、`services.gradle.org` 可能不可达；无 sudo 装 JDK 17；磁盘空间也可能不足。
- 沙箱首次 `npm install` 慢，45 秒 timeout 可能被砍；可用 `npm install --no-audit --no-fund --silent`，失败重试 1-2 次。
- `/tmp` 不持久；`/tmp/mimi-test`、`/tmp/mimi-build` 这类目录不能当长期状态。

## 5-7. Android / JDK / 依赖平台

- Capacitor 6 的 Gradle 插件需要 JDK 17 字节码；JDK 21+ 可能报 `Unsupported class file major version 65`。
- `ANDROID_HOME` 不设会报 `SDK location not found`。

```cmd
ANDROID_HOME=%LOCALAPPDATA%\Android\Sdk
```

或写 `android/local.properties`：

```properties
sdk.dir=C\:\\Users\\zlbdh\\AppData\\Local\\Android\\Sdk
```

Windows `node_modules` 含 Windows 二进制，例如 `@rolldown/binding-win32-x64`；Linux 沙箱要自己装 Linux 依赖。

## 8-10. Storage / CSS / 配额

- 老 v1 使用过 Claude Artifact 专属 `window.storage.get/set/delete`；当前由 `src/storage-shim.js` 在 `main.jsx` 前加载并映射到 localStorage。新代码不要再用 `window.storage`，直接用 Dexie / IndexedDB。
- CSS 字体 `@import` 必须在 `@tailwind base` 前，否则 PostCSS 警告并失效。
- localStorage 约 5MB；聊天历史、备份等大数据进入 IndexedDB，localStorage 只放 API key、budget、设置标记等少量配置。

## 11. `npm.ps1` shim 参数错位

本机 `npm` 可能解析到 `D:\Program Files\nodejs\npm.ps1`。PowerShell 里 `& npm test` 会把参数弄错，npm 收到 `"pm"` 并报 `Unknown command: "pm"`。

防御：

- 脚本里用 `cmd /c "npm <子命令>"`。
- 或直接用 `& npm.cmd <子命令>`。

2026-06-14 发布门禁 hook 首次踩中，被 ADR-033 真机整跑抓出并修复。

## 12. PS5.1 `2>&1` + EAP=Stop 误终止

Windows PowerShell 5.1 中，`& 原生命令 ... 2>&1` 把 stderr 并进输出，同时脚本顶部 `$ErrorActionPreference="Stop"` 时，原生命令只要往 stderr 写任何一行，PS 就可能包成 `NativeCommandError` 并提前终止。

表现：原生命令退出码实际为 0，脚本却误红。

防御：捕获输出时临时降 EAP，真实成败只认 `$LASTEXITCODE`；只判成败时可不并 stderr。

此问题在 2026-06-14 `hooks-smoke.ps1` 首次真跑中暴露：vite 良性 deprecation warning 进 stderr，smoke 误红。已在 hooks smoke 支撑层回归覆盖。
