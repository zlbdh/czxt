# 规则：已知技术约束（12 项速查）

这些是项目反复踩过的坑。起手先看本入口；现象、历史案例和命令模板见 [`已知技术约束-附录.md`](已知技术约束-附录.md)。

## 先记住 4 条

1. **大文件改动先看 P4b**：接近 6500B 的文件要谨慎，>=8000B 先考虑拆分；必须改大文件时，改完读回大小和尾部。
2. **APK 以本地 Windows `build-apk.ps1` 为主，GitHub Actions 手动触发为备用**：沙箱通常无 Android SDK / 网络 / JDK 条件，不把“沙箱不能打 APK”误判为业务失败；出包由测试发布 PM按闭环权限承接。
3. **Android/Capacitor 固定 JDK 17**：不用 JDK 21+ 直接跑 Gradle。
4. **Windows PowerShell 调原生命令要防 shim / stderr 误判**：脚本里优先 `npm.cmd` 或 `cmd /c "npm ..."`；捕获 stderr 时真实成败看 `$LASTEXITCODE`。

## 12 项约束总表

| # | 约束 | 触发场景 | 处理动作 |
|---|---|---|---|
| 1 | Mount / 大文件写入截断风险 | 改 `操作系统/`、`能力资产/`、`{{APP_REPO_DIR}}/src/` 大文件 | P4b 先看；>6500B 谨慎，>=8000B 优先拆；改完读回大小和尾部 |
| 2 | 沙箱无 Android SDK + 无法访问 Google/Gradle 源 | 要出 APK / Gradle 构建 | 交 Windows 真机环境；沙箱只跑代码、vitest、vite build |
| 3 | 沙箱 `npm install` 慢 | 首次装依赖 | 用 `--no-audit --no-fund --silent`，失败重试，必要时 `--prefer-offline` |
| 4 | `/tmp` 沙箱重启会清 | 临时构建 / 测试目录 | 不依赖 `/tmp` 持久；产物要拷回项目路径 |
| 5 | Capacitor + Android 必须 JDK 17 | `gradlew assembleDebug` | `JAVA_HOME` 指向 Eclipse Temurin 17，不用最新版 JDK |
| 6 | `ANDROID_HOME` 必须设 | Android 构建 | 设 `%LOCALAPPDATA%\Android\Sdk` 或写 `android/local.properties` |
| 7 | `node_modules` 跨平台不通用 | Windows 依赖拿到 Linux 沙箱跑 | 沙箱自己装 Linux 依赖，不复用 Windows `node_modules` |
| 8 | `window.storage` 只是兼容层 | 迁移老 v1 / Claude Artifact 代码 | 老代码靠 shim；新代码直接 Dexie/IndexedDB |
| 9 | CSS `@import` 必须在顶部 | 改 `src/index.css` 字体 | `@import` 放第 1 行，再写 `@tailwind` |
| 10 | localStorage 配额约 5MB | 聊天历史 / 备份 / 大数据 | 大数据进 IndexedDB，localStorage 只存少量配置 |
| 11 | PowerShell 里 `npm.ps1` shim 参数错位 | 脚本中跑 npm 子命令 | 用 `npm.cmd` 或 `cmd /c "npm <子命令>"` |
| 12 | PS5.1 `2>&1` + EAP=Stop 会误终止 | 脚本捕获原生命令 stderr | 临时降 `$ErrorActionPreference="Continue"`，再判 `$LASTEXITCODE` |

## 大文件写入当前口径

| 工具 / 场景 | 阈值 | 行为 |
|---|---|---|
| Cowork / mount 截断风险 | 6500B 软警戒，约 9KB 历史硬截断 | 用脚本写入或拆分，改完读回验证 |
| Claude Code / Codex | 8000B 工程软建议 | 优先拆成入口 + 附录；不得为了降字节丢契约 |
| 所有工具 | 关键判断前 | 信 `check-operating-system.ps1` / P4b / Read 复核，不信单独 `wc -c` |

## PowerShell 原生命令安全模板

```powershell
$prevEAP = $ErrorActionPreference
$ErrorActionPreference = "Continue"
try {
  $output = & $cmd @args 2>&1
  $code = $LASTEXITCODE
} finally {
  $ErrorActionPreference = $prevEAP
}
if ($code -ne 0) { throw "failed ($code): $output" }
```
