## 一、AI 模型层（事实，不是猜测）

| 项 | 真实状态 |
|---|---|
| **AI 模型** | **小米自研 MiMo**（`mimo-v2.5-pro`）— **国内合规模型** ⭐ |
| **服务商** | **小米官方**（不是第三方中转） |
| **API 协议** | Anthropic 兼容（`/v1/messages` + `x-api-key`）— 协议层包装，方便未来切换其他 AI 解耦 |
| **baseUrl** | 以 `{{APP_REPO_DIR}}/src/shared/ai.js` 的 `LLM_DEFAULTS.baseUrl` 为准；当前默认 `https://token-plan-cn.xiaomimimo.com/anthropic` |
| **协议端点** | `/v1/messages`（POST）— 与 Anthropic 完全一致 |
| **认证 header** | `x-api-key: <key>` + `anthropic-version: 2023-06-01` |
| **官方网站** | `https://xiaomimimo.com` |

### 1.1 重要事实（绝对不能误读）

- ✅ **xiaomimimo.com = 小米官方服务**（不是第三方聚合）
- ✅ **mimo-v2.5-pro = 小米自研 MiMo 大模型**（不是包装 Claude / GPT）
- ✅ **国内合规** — A 阶段公开发布友好（不会撞海外模型的合规墙）
- ✅ **Anthropic 协议是包装层** — 代码用 `/v1/messages` 接口，但实质跑的是 MiMo

### 1.2 协议层 vs 模型层的区别（防误解）

```
代码层：fetch('https://token-plan-cn.xiaomimimo.com/anthropic/v1/messages')
       ↓（Anthropic 协议兼容）
小米路由：识别 model: 'mimo-v2.5-pro' → 路由到小米 MiMo
       ↓
实际推理：小米自研 MiMo v2.5 Pro 模型
       ↓
返回：Anthropic 协议格式 JSON
```

**协议 = Anthropic 兼容**（方便代码用 Claude SDK 风格调用）。
**模型 = 小米 MiMo**（实质推理是 MiMo）。
两者**不冲突** — 这是常见的 API 包装模式。

## 二、内容创作时（运营咪咪重点）

### 2.1 ✅ 可以这样说
- 「咪咪用了**小米自研 MiMo 大模型**」
- 「跑在小米官方 API 上」
- 「小米官方服务；数据驻留 / 是否出境待 zlbdh confirm，公开口径暂不承诺」

### 2.2 ❌ 绝对不要这样说
- ❌「咪咪用了 Claude」（错 — Claude 是 Anthropic 的，我们只是用了它的协议）
- ❌「咪咪用了 OpenAI / GPT」（错）
- ❌「咪咪用了第三方 API 中转」（错 — 是小米官方直连）
- ❌「我也不知道用的什么模型」（错 — 知道是小米 MiMo）

### 2.3 用户混淆点（如有人问起）

| 用户可能问 | 标准回答 |
|---|---|
| 你用的是 Claude 吗？| 不，是小米自研 MiMo。但代码层兼容 Anthropic 协议（这是个常见的 API 包装模式）|
| 跑在国外服务器？| 当前接入小米官方服务；数据驻留 / 是否出境待 zlbdh confirm，公开口径暂不承诺 |
| 是不是套壳？| 不是。小米官方服务 + 小米自研 MiMo 模型 + 协议层是 Anthropic 兼容（解耦设计）|
