## 推断 1：APK + smoke 是否完成（解决问题 #1，PROP-007 P1-2 强化）

**逻辑**：`{{APP_REPO_DIR}}/apk/` 最新文件时间戳 vs 当前需求清单 / 需求历史中所有「⏳ 待 APK」F-XXX 的代码完成时间。
如果最新 APK > 任何「⏳ 待 APK」F 的代码完成时间 → 主动问"是不是 smoke 过了 F-XXX？"

**触发式强化**（PROP-007 后按当前响应式边界收口）：
- session 起手、发布闭环、看到 `待 APK / 待 smoke`、用户主动要求状态推断时必查
- 普通对话不强制每次响应都跑；直到该建议被 dismiss 或确认完成（参考 状态.md 决议表）
- 主动问：「我注意到 F-XXX 还在 ⏳，最新 APK 是 N 天前的 — 是不是已经测过了？」
- 用户回 OK → 由对应 PM 按路径白名单、状态规则和留痕要求确认后更新；回「还没」→ 7 天 cooldown 不重提

```powershell
# Windows/Codex 权威入口：先切到当前项目根
Set-Location "{{PROJECT_ROOT}}"

# 最新 APK 时间戳
$latestApk = Get-ChildItem -LiteralPath "{{APP_REPO_DIR}}\apk" -Filter "*.apk" -File |
  Sort-Object LastWriteTime -Descending |
  Select-Object -First 1
$latestApk | Select-Object FullName, LastWriteTime

# 当前需求清单中「⏳ 待 APK」F-XXX（按所有 Sprint 清单扫，不硬编码 Sprint-1）
Select-String -Path "Docs\1-需求文档\Sprint-*需求清单.md" -Pattern "^\| F-[A-Z0-9-]+.*⏳"
```

**输出建议**：
```
💡 检测到 {{APP_REPO_DIR}}/apk/{{PROJECT_NAME}}-vX.Y.Z-debug.apk（YYYY-MM-DD），但当前需求清单中：
   - F-XXX 状态：⏳ 待 APK
是不是已经装机 + smoke 过了？要不要切 ✅？
```

## 推断 2：代码改动反映在文档（解决问题 #11）

**逻辑**：{{APP_REPO_DIR}}/src 最近改动的文件 mtime vs 需求历史.md 最后一条。
如果代码 mtime > 需求历史最后一条时间 + 1 天 → 主动问"代码动了，需求历史要不要更新？"

```bash
# {{APP_REPO_DIR}}/src 最新文件
latest_code=$(find {{APP_REPO_DIR}}/src -type f \( -name '*.jsx' -o -name '*.js' \) -printf '%T@ %p\n' | sort -rn | head -1)

# 需求历史最后一条日期（粗略 grep 最近 ✅）
grep -E "✅.*已完成（[0-9]{4}-[0-9]{2}-[0-9]{2}" Docs/1-需求文档/需求历史.md | tail -1
```

**逻辑**：每个 ADR 头部「状态」字段 vs ADR README 索引表里的对应行，应该一致。

```bash
for f in Docs/3-开发文档/adr/ADR-*.md; do
  num=$(basename "$f" | grep -oE 'ADR-[0-9]+')
  head_status=$(head -10 "$f" | grep "状态" | head -1)
  index_status=$(grep -a "^| $num " Docs/3-开发文档/adr/README.md)
  # 比对
done
```

**输出建议**：
