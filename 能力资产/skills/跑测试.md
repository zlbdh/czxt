# 技能：跑测试

每次大改后必跑。看 `Docs/4-测试文档/测试策略.md` 是策略，这里是具体命令。

## Windows / Codex 当前推荐

核心 QA：

```powershell
cd {{PROJECT_ROOT}}\{{APP_REPO_DIR}}
npm test -- --run
npm run build
```

Android / APK 前准备再继续跑：

```powershell
npx cap sync android
```

需要 APK 时继续跑：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\build-apk.ps1
```

下面的 `/tmp/mimi-*` 沙箱跑法仅作 Cowork 旧环境兜底。

## 一键跑全套

```bash
WS="/sessions/<sandbox>/mnt/{{PROJECT_NAME}}/{{APP_REPO_DIR}}"
T=/tmp/mimi-test
B=/tmp/mimi-build

# 同步源到测试沙箱
cp -rf "$WS/src/." "$T/src/"
cp -rf "$WS/src/." "$B/src/"

# 跑语法
cd "$T" && node syntax-check.mjs

# 跑 vitest
cd "$T" && npx vitest run

# 跑 vite build
cd "$B" && npx vite build
```

## 沙箱第一次设置

```bash
T=/tmp/mimi-test
B=/tmp/mimi-build
mkdir -p "$T/src" "$B/src" "$B/public"

cd "$T"
cat > package.json <<'EOF'
{
  "name": "mimi-test",
  "type": "module",
  "scripts": { "test": "vitest run" },
  "devDependencies": { "vitest": "^4.1.5", "esbuild": "^0.25.0" }
}
EOF
cat > syntax-check.mjs <<'EOF'
import { transformSync } from 'esbuild';
import { readFileSync, readdirSync, statSync } from 'fs';
import { join } from 'path';

function walk(dir, files = []) {
  for (const name of readdirSync(dir)) {
    if (name === 'node_modules') continue;
    const p = join(dir, name);
    if (statSync(p).isDirectory()) walk(p, files);
    else if (/\.(js|jsx)$/.test(name)) files.push(p);
  }
  return files;
}
let ok = 0, fail = 0;
for (const f of walk('/tmp/mimi-test/src').sort()) {
  try {
    transformSync(readFileSync(f, 'utf8'), {
      loader: f.endsWith('.jsx') ? 'jsx' : 'js',
      target: 'es2020'
    });
    ok++;
    console.log(`  ✓ ${f.replace('/tmp/mimi-test/', '')}`);
  } catch (e) {
    fail++;
    console.log(`  ✗ ${f.replace('/tmp/mimi-test/', '')} → ${(e.errors?.[0]?.text || e.message).slice(0, 100)}`);
  }
}
console.log(`\n${ok} OK / ${fail} fail`);
EOF
timeout 38 npm install --no-audit --no-fund --silent

# 同样设置 build 沙箱
cp -rf "/sessions/<sandbox>/mnt/{{PROJECT_NAME}}/{{APP_REPO_DIR}}"/{package.json,vite.config.js,tailwind.config.js,postcss.config.js,index.html} "$B/"
cp -rf "/sessions/<sandbox>/mnt/{{PROJECT_NAME}}/{{APP_REPO_DIR}}/public/." "$B/public/"
cd "$B"
timeout 38 npm install --no-audit --no-fund --silent
```

## 失败排查

### 语法 fail
- 看 line number
- mount 截断？跑 `wc -lc` + `tail -3`
- import 名错？grep 找
- JSX 标签未闭合？

### vitest fail
- 看测试名 + expected vs received
- 是新功能 break 旧测试？→ 修代码或测试（看 AC）
- 是新加测试本身错？→ 修测试

### vite build fail
- 看 `Module not found` → 加 import
- 看 `Cannot find package` → npm install 漏装
- 看 `Top-level await` 等语法警告 → 改 target

## 加新测试

新建 `src/shared/xxx.test.js`：

```js
import { describe, expect, it } from 'vitest';
import { fnUnderTest } from './xxx.js';

describe('fnUnderTest', () => {
  it('handles happy path', () => {
    expect(fnUnderTest(input)).toEqual(expected);
  });

  it('handles edge: null', () => {
    expect(fnUnderTest(null)).toBe(null);
  });

  it('handles edge: empty', () => {
    expect(fnUnderTest('')).toBe(null);
  });
});
```

跑：
```bash
cd /tmp/mimi-test
cp /sessions/<sandbox>/mnt/{{PROJECT_NAME}}/{{APP_REPO_DIR}}/src/shared/xxx.test.js src/shared/
npx vitest run src/shared/xxx.test.js
```

## 别跳过

- 不跳过：「只是改了样式」也跑 vitest（可能 import 错）
- 不跳过：「上次 build 过了」也跑 vite build（依赖可能新增）
- 不跳过：「测试都过了」也跑 vite build（生产模式更严格）
