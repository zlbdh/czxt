# 项目体检 — 检查项 1-4 附录

## 检查项 1：源文件大小命令

> 低频 Bash 粗扫只作候选，以 `check-operating-system.ps1` P4b 为准；P4b 会排除 `状态-archive`、`CHANGELOG-2026*`、`历史归档/`、`agent-*-历史.md`。

```bash
cd "<项目根>"

# Scope: 仅 {{APP_REPO_DIR}}/src + 操作系统/ + 能力资产/
find {{APP_REPO_DIR}}/src 操作系统 能力资产 -type f \( -name '*.jsx' -o -name '*.js' -o -name '*.ts' -o -name '*.tsx' -o -name '*.md' -o -name '*.ps1' -o -name '*.json' \) \
  | grep -Ev '状态-archive|CHANGELOG-2026|历史归档|agent-(INDEX|README)-历史\.md' \
  | xargs -r wc -c \
  | awk '$1>=6000 && $2!="total" {print $1" "$2}' | sort -rn
```

关键决策点必须双源复核：

1. 跑 bash `wc -c` 找候选。
2. 对 >=6500B（Cowork）或 >=8KB（Claude Code/Codex）文件用 Read 复核。
3. bash 与 Read 不一致时信 Read。

历史拆分：2026-05-11 PROP-008 / ADR-011 触发的 6 个 >=6500B 文件已拆完，见 [ADR-011](../../Docs/3-开发文档/adr/ADR-011-6500B文件拆分.md)；PROP-013 后仍保留工具分层阈值。

## 检查项 2：PRD 双源一致性命令

```bash
cd "<项目根>"

grep -nE "已完成.*(待 |TBD|⏳|待写|待 smoke|待 APK)" Docs/1-需求文档/Sprint-*需求清单.md
```

输出应为空；若有匹配，说明状态字段没有对齐。若代码已完成但 APK / smoke 仍等待，Sprint 顶部表格状态列也应标为等待态。

## 检查项 4：旧路径残留命令

> 权威守卫是 `check-operating-system.ps1` 的 P4j/P4k；下面只是人工解释模板。历史归档、`agent-*-历史.md`、当前 `能力资产/rules/` 相对链接和本文件自身的说明不算残留。

```bash
cd "<项目根>"

grep -aPn "agent/|Agent/|(?<!能力资产/)rules/" \
  README.md 操作系统 能力资产 Docs/3-开发文档/项目结构.md Docs/3-开发文档/adr/README.md \
  Docs/7-复盘/README.md Docs/7-复盘/_模板.md \
  确认改动/README.md 确认改动/_模板.md 2>/dev/null \
  | grep -vE "历史归档|状态-archive|CHANGELOG-2026|agent-(INDEX|README)-历史\.md|项目体检-检查项-1-4-附录\.md" \
  | grep -v "Agent/ 整合到 rules/\|agents/skills/workflows/rules" \
  | grep -v "│   ├── rules" \
  | grep -vE ":[0-9]+:\| ADR-[0-9]+ \|"
```

判断模板：

```bash
out=$(...上述 grep 链...)
[ -z "$out" ] && echo "✅ 通过" || printf '🔴 残留：\n%s\n' "$out"
```

历史档案中的 `rules/...`、`Agent/agents/`、`Agent/skills/`、`Agent/workflows/` 可以保留。它们是当时迁移映射表和背景描述，不代表当前事实源；当前能力资产自己的 `rules/` 相对链接也不是旧路径残留。
