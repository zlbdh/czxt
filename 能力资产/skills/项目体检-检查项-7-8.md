## 检查项 7：framework 引用频率监控（PS1 P4c）

### 病灶

`操作系统/` + `能力资产/` 内 .md 容易随演进只加不删，P4c 用引用频率找死代码和孤儿候选。

### 阈值分级

| 引用次数 | 级别 | 含义 | 动作 |
|---|---|---|---|
| >= 10 | ⭐ 高活跃 | framework 主干，多处依赖 | 保留 |
| 5-9 | ✅ 中频 | 稳定参考 | 保留 |
| 2-4 | 🟢 低频 | 边缘但仍有用 | 监控，下次 RETRO 议题 |
| 1 | 🟡 极低频 | 孤儿候选 | 评估整合 |
| 0 | 🔴 死代码 | 没人引用 | 删除或修复链接（PROP 治理） |

### 扫描范围

活跃 `操作系统/**/*.md` + `能力资产/**/*.md`；引用源覆盖 framework、Docs、PROP、交接区、PM 工作区和根入口。按 filename 出现在其他 .md 的文件数统计，同一引用文件不重复累加。

### 权威命令

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File 能力资产/tools/scripts/check-operating-system.ps1
# 看【P4c】framework 引用频率体检

# 需要人工审计 Low 2-4 文件时，单跑 P4c 并展示低频清单
powershell -NoProfile -ExecutionPolicy Bypass -File 能力资产/tools/scripts/check-os/p4c-framework-references.ps1 -ShowLowList
```

### 判定

- ✅ 0 死代码 + 0 极低频最健康。
- 🟡 极低频必须评估；Low 2-4 用 `-ShowLowList` 抽查，优先判断自然入口是否缺失，不为计数硬刷链接。
- 🔴 >= 1 个死代码，必须处理：删除、改路径或修复链接。

## 检查项 8：状态.md 30 天新鲜度（PS1 P4d）

### 病灶

`状态.md` 是跨 session 快照中枢。P4d 做双维度对账：

| 维度 | 目的 |
|---|---|
| 文件系统 mtime | 看文件最近是否被真实改过 |
| 内容最新 `YYYY-MM-DD` 日期戳 | 防 git checkout / clone 重置 mtime 但内容仍旧 |

### 阈值

| 维度 | <= 30 天 | > 30 天 |
|---|---|---|
| 文件 mtime | ✅ | 🟡 stale |
| 内容最新日期戳 | ✅ | 🟡 内容 stale |

任一维度超阈值即 stale；不阻塞，但新 session 起手必须先修状态或走状态推断。

### 权威命令

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File 能力资产/tools/scripts/check-operating-system.ps1
# 看【P4d】状态.md 新鲜度
```

### 判定

- ✅ 两维度均 <= 30 天。
- 🟡 任一维度 > 30 天，提醒项目 PM 更新。
- 🔴 文件缺失，P4a 基础完整性已经触发。

归档例外：如果项目真的 30 天未动，`状态.md` 内容应明示“项目已封存 / 暂停”，并在 P4d 输出后人工 dismiss。

## 检查项 9：P4e mount 缓存陷阱提醒（议题 D）

### 病灶

Cowork 通过 Linux mount 读 Windows host 文件时，`wc -c` / `ls -la` / `stat` 可能读到 stale 字节数。刚写完就用 bash 判大小，容易误判。

### 四条防御规则

| # | 规则 | 触发场景 |
|---|---|---|
| ① | Read 工具复核警戒 / 危险文件大小，不信 bash `wc -c` | 接近 6500B / 8KB |
| ② | Edit / Write / apply_patch 改 framework 文件后立即 Read 复核 tail | 验证写入实际生效 |
| ③ | git status 不一致时刷新 mount 或重启 Cowork session | bash 看不到新文件 / 看到旧 diff |
| ④ | P4b 警戒清单字节数信 PS1，不信 Cowork bash | 双方数字不一致时 |

### 何时 bash 可信

- ✅ 只读现有内容且不关心字节精度：grep、找关键字、跨文件统计。
- ✅ 跨 session 已经过几分钟，mount 缓存自然过期。
- ❌ 刚刚 Edit / Write / apply_patch 后立即 `wc -c`。
- ❌ 用作是否超警戒线的决策依据；此时信 PS1 + Read。

### 权威命令

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File 能力资产/tools/scripts/check-operating-system.ps1
# 看【P4e】Mount 缓存陷阱提醒
```

P4e 是静态提醒，不阻塞；它是跨 session 认知护栏，不是自动修复器。
