## P4c Bash 兜底（粗略估算）

> 仅作旧环境兜底，不等价于 PS1 P4c；权威 P4c 会排除 `历史归档`、`状态-archive`、`CHANGELOG-2026` 并按文件名出现在“其他文档”的文件数统计。

```bash
# 先切到当前项目根；Windows/Codex 为 {{PROJECT_ROOT}}
# Cowork 为 /sessions/<sandbox>/mnt/{{PROJECT_NAME}}
cd "<项目根>"

for f in $(find 操作系统 能力资产 -name '*.md' -type f | grep -Ev '历史归档|状态-archive|CHANGELOG-2026'); do
  name=$(basename "$f")
  count=$(grep -l "$name" \
    $(find 操作系统 能力资产 Docs 确认改动 交接区 PM工作区 -name '*.md' -type f | grep -Ev '历史归档|状态-archive|CHANGELOG-2026') \
    AGENTS.md 状态.md README.md 2>/dev/null \
    | grep -v "$f" | wc -l)
  echo "$count $f"
done | sort -n | head -10
```

历史背景：PROP-017 反向学习自 `D:\WGKJ\账号管理`。当时扫描发现 6 个 0 引用 `.md`，修复后 framework 健康度明显提升，因此把引用频率纳入 P4c。

## P4d Bash 兜底

```bash
cd "<项目根>"

# 维度 1：mtime
now=$(date +%s)
threshold=$(date -d '30 days ago' +%s)
mtime=$(stat -c %Y 状态.md)
days=$(( (now - mtime) / 86400 ))
[ "$mtime" -lt "$threshold" ] && echo "🟡 mtime stale ($days 天)" || echo "✅ mtime $days 天"

# 维度 2：内容最新日期戳
latest=$(grep -oE '[0-9]{4}-[0-9]{2}-[0-9]{2}' 状态.md | sort -ru | head -1)
echo "内容最新日期: $latest"
```

注意：P4d 是 stale 提醒，不是业务状态证明。`状态.md` 自身也声明它只是快照，不是真相；需要结合最新交接卡和项目真源文件判断。

## P4e 历史背景

议题 D 来自 Cowork mount stale 的多次实战暴露。典型表现：

- Windows host 已改文件，Cowork bash 仍读旧字节数。
- bash 输出看似精确，实际比 Read / PS1 慢半拍。
- 结果会误导“是否拆分”“是否超 6500B”“是否写入成功”等关键决策。

已记录的高信号案例：

- RETRO-005 卡 2：`Health.jsx` 缓存读到旧字节。
- PM 自纠 #9 / #12：PS1 同步失败或警告差异。
- PM 自纠 #23 / #25：F-PREP-1 字节差 -5129B / -2913B。
- PM 自纠 #26 / #32 / #36：F-BRIEFING-1 / F-DEVIATION-2 字节差 -4111B / -2913B 等。
- PROP-018 P1：CHANGELOG 拆分后用 Read 复核验证写入。

---

## 为什么不自动 bash vs PowerShell 对比

- PS1 在 Windows host 跑，无法直接读 Cowork bash 视角。
- stale 属 mount 协议和工具缓存行为，跨工具同步策略不同。
- 自动对比会引入复杂状态和误报；MVP 选择“认知提醒 + 防御流程”。

因此 P4e 设计成每次体检都输出的静态提醒：不阻塞，但让 PM 不再靠记忆躲坑。

## P4o Bash 兜底（粗略估算）

> 仅作旧环境兜底，不检查锚点；权威 P4o 是 PowerShell 脚本，会排除历史归档并校验 Markdown heading slug。

```bash
cd "<项目根>"

python - <<'PY'
import os, re, urllib.parse
root=os.getcwd()
scopes=['操作系统','能力资产']
bad=[]
for scope in scopes:
  for d, dirs, files in os.walk(os.path.join(root, scope)):
    dirs[:] = [x for x in dirs if x not in ['历史归档','状态-archive']]
    if '历史归档' in d or '状态-archive' in d: continue
    for name in files:
      if not name.endswith('.md') or name.startswith('CHANGELOG-'): continue
      p=os.path.join(d,name)
      text=open(p,encoding='utf-8',errors='replace').read()
      for m in re.finditer(r'!?\[[^\]]*\]\(([^)]+)\)', text):
        raw=m.group(1).strip()
        if not raw or raw.startswith('#') or re.match(r'^[a-z][a-z0-9+.-]*:', raw, re.I): continue
        target=urllib.parse.unquote(raw.split('#',1)[0]).replace('/', os.sep)
        if target and not os.path.exists(os.path.normpath(os.path.join(d,target))):
          bad.append((os.path.relpath(p,root), text.count('\n',0,m.start())+1, raw))
for row in bad[:50]: print(*row, sep=':')
raise SystemExit(1 if bad else 0)
PY
```

## 信源信任度

当 bash、Read、PS1 互相冲突时，当前操作系统倾向：

1. Windows host PS1 体检输出。
2. Read 工具或等价的真实文件读取。
3. bash grep 这类内容检索。
4. bash `wc -c` / `stat` 在刚写入后的结果。

这个顺序只用于文件同步和大小判断，不替代业务测试、真机 smoke 或发布门禁。
