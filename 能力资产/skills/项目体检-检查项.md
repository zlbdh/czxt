## 检查项 1 / 9：源文件大小（按工具分层，PROP-013 / ADR-017 后）

### 阈值分级（按工具分层）

**Cowork（mount 截断风险）**：

| 字节区间 | 状态 | 动作 |
|---|---|---|
| < 6000B | ✅ 安全 | Edit / Write 都可用 |
| 6000-6500B | 🟡 警戒 | Edit 前先 Read 复核；改大文件建议 Python/Bash 写入 |
| 6500-8000B | 🟡 软建议 | Cowork 改这些文件**必须用 Python/Bash 绕 Edit**；不强制开 PROP |
| ≥ 8000B | 🔴 危险（接近 mount 9KB 硬截断）| Cowork 不要用 Edit；交 Claude Code / Codex 改 |

**Claude Code / Codex（本地文件系统）**：

| 字节区间 | 状态 | 动作 |
|---|---|---|
| < 8KB | ✅ 安全 | 无限制 |
| ≥ 8KB | 🟡 软建议抽组件 | 工程审美层面建议拆（单一职责），**不阻塞实施** |

**为什么按工具分层（PROP-013 / ADR-017）**：6500B 门槛源自 Cowork mount 截断风险；Claude Code / Codex 无该写入限制，只保留工程审美层面的 8KB 软建议。

### Scope 明示（PROP-011 后体检 v26 加）

| 范围 | 适用？ | 理由 |
|---|---|---|
| `{{APP_REPO_DIR}}/src/` | ✅ 适用 | 业务代码，高频 Edit，mount 截断风险高 |
| `操作系统/` | ✅ 适用 | 操作系统规则层，AI 频繁读写 |
| `能力资产/` | ✅ 适用 | hooks / skills / rules / tools 执行层，AI 频繁读写 |
| `PM工作区/` | ❌ 不按大小扫 | 只查入口状态、9 PM 目录/README 和旧口径；草稿、HTML、图片、PM 自纠历史不按 P4b 大小阻断 |
| `Docs/` | ❌ 豁免 | 业务 PRD / 复盘 / ADR / 历史归档 — 写完很少改 |
| `确认改动/已审批/已完成/` | ❌ 豁免 | PROP 档案，写完不动 |
| `交接区/已接手/` + `**/历史归档/` + `agent-*-历史.md` | ❌ 豁免 | 历史档案 |
| `交接区/待接手/` | 🟡 边界 | 短期文件，建议 < 6500B 但不强制 |

→ 体检命令仅扫活跃文件：`{{APP_REPO_DIR}}/src` + `操作系统` + `能力资产` 的 `.js/.jsx/.ts/.tsx/.md/.ps1/.json`，并排除 `状态-archive` / `CHANGELOG-2026*` / `历史归档/` / `agent-*-历史.md`。

### 日常判定

- 当前结果以 `check-operating-system.ps1` P4b 输出为准；P4b 会给出 `{{APP_REPO_DIR}}/src`、`操作系统`、`能力资产` 分域摘要。
- Bash `wc -c` 只做候选扫描；关键决策信 Windows host PS1 + Read 复核。
- 低频 bash 扫描命令和历史拆分说明见 [`项目体检-检查项-1-4-附录.md`](项目体检-检查项-1-4-附录.md)。

## 检查项 2 / 9：PRD 双源一致性

### 病灶

PRD 文件常含「表格 + 子段标题」双层状态描述。Claude Code 收工时容易只改表格不改子段。

### 日常判定

- Sprint 顶部表格与子段标题必须同态。
- “已完成”段不应残留“待 X / TBD / ⏳”等字眼。
- 人工 grep 模板见 [`项目体检-检查项-1-4-附录.md`](项目体检-检查项-1-4-附录.md)。

### 例外
- F-XXX 段如果**真的还在等 APK / smoke**（代码完了但 APK 还没出），可以保留「待 APK + 真机 smoke」字眼——但要在 Sprint 顶部表格的状态列同时标 ⏳ 而不是 ✅。

## 检查项 3 / 9：PROP / ADR / RETRO 计数对齐

### 检查规则

- PROP 计数 = 待审批 + 进行中 + 已完成 + 已弃用 + 拒绝
- ADR 编号连续无跳号
- RETRO 文件/README 一致性由 P4i/readme-index 守；沉淀节奏跑 `能力资产/tools/scripts/check-retro-cadence.ps1`。业务 L3+ 数量只作人工参考，不再用固定公式自动推导 RETRO 编号。

### 比对要点

- 「确认改动/README.md」表格里的计数 vs 实际目录里的文件数 应该一致
- ADR README 索引表的状态字段 vs 各 ADR 文件头的状态字段 应该一致
- 自动守卫：`能力资产/tools/scripts/check-readme-indexes.ps1` 与 `能力资产/tools/scripts/check-operating-system.ps1` 会核对 PROP 五列计数、ADR README 行数和元规则池计数；人工命令只用于解释差异来源。
- 低频解释命令见 [`项目体检-检查项-1-4-附录.md`](项目体检-检查项-1-4-附录.md)。

## 检查项 4 / 9：旧路径残留（防 ADR-007 后回潮）

### 日常判定

- 活文档不应再有旧顶层 `rules/` 或旧 `agent/` 当前事实源引用。
- 历史描述、迁移映射表和归档文件可保留当时路径。
- 人工 grep 模板见 [`项目体检-检查项-1-4-附录.md`](项目体检-检查项-1-4-附录.md)。

### 历史档案不动
- 历史 PROP/ADR/RETRO 内文里的 `rules/...` 链接是写下时刻快照，按 ADR-007 §C 不动
- ADR-003 / ADR-007 中的 `Agent/agents/`、`Agent/skills/`、`Agent/workflows/` 是**迁移映射表**和**历史背景描述**（描述 ADR-003/004/007 当时干了什么），合法保留
- 体检 skill 自身包含旧路径 grep 文本时，只能作为模板说明，不作为当前事实源。
