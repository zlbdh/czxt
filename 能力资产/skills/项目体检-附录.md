## 维度 6 低频命令

> 旧环境 Bash/GNU 示例，仅适用于 WSL / Git Bash；Windows/Codex 优先跑 `能力资产/tools/scripts/check-handoff-zone.ps1`。

```bash
cd "<项目根>"

# 待接手卡数
handoff_count=$(ls 交接区/待接手/*.md 2>/dev/null | wc -l)
echo "待接手卡数: $handoff_count"
[ "$handoff_count" -gt 3 ] && echo "  🟡 积压（≤3 才安全）"

# 已接手 > 30 天
threshold=$(date -d '30 days ago' +%s)
for f in 交接区/已接手/*.md; do
  [ -f "$f" ] || continue
  mtime=$(stat -c %Y "$f")
  [ "$mtime" -lt "$threshold" ] && echo "  🟡 $f 超 30 天，建议归档"
done

# 状态.md 摘要对账
echo "用 Read 读 状态.md 顶部摘要段 vs 交接区/待接手/最新文件"
```

## 历史说明

项目体检最初是人工技能，后来逐步把高风险项下沉为 `check-operating-system.ps1` 的 P4 自动守卫。当前人工文档的职责是解释“为什么”和“失败后怎么判”，不复制每个脚本的完整实现。

维护原则：

- 主文只保留日常入口和总表。
- 检查项文件保留判定契约。
- 附录承接低频命令、历史背景、报告模板。
- 真正的当前结果以 PS1 输出为准。
