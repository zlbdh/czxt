# 技能：项目体检

定期跑一遍，找出隐藏问题：大文件、PRD 双源不一致、计数不齐、旧路径残留、跨 session 同步遗漏、交接区积压、framework 死引用、状态陈旧、mount 缓存误判和活跃 Markdown 断链。

本文件只保留高频入口。报告模板和低频命令见 [`项目体检-附录.md`](项目体检-附录.md)。

## 何时跑

- 每次 PROP 收档前必跑，详见 [`实施循环.md`](../../操作系统/07_完整工作流/实施循环.md)。
- zlbdh 主动要求跑项目体检。
- 重大 framework / hooks / 工作流改动后。
- 长 session 末尾自查。
- Cowork / Claude Code / Codex 等运行时切换时，防信息差。

## 权威入口

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File 能力资产/tools/scripts/check-operating-system.ps1
```

当前脚本为 façade + `check-os/` 子模块结构，覆盖 P4a-P4s。P4b 会输出 `{{APP_REPO_DIR}}/src` 业务 P4b 治理摘要，把大文件债按测试/生产和目录域分组，并强调 owner=开发 PM「实施者」、按功能触发拆、不为数字清零；P4o 同时通过 `check-readme-indexes.ps1` 进入 PostToolUse 快检链，改 framework/PM 工作区后可自动提醒活跃 Markdown 断链；P4p 守住 shared / mcp / agents / workflows 的旧口径回潮；P4q 守住治理语义锚点与 PROP 状态机；P4r 守住 hooks 配置与 scheduled/watch 运行态锚点；P4s 守住模板根纯净度，防具体项目卡和来源项目关键词回流。AGENTS 起手口径里的“项目体检”= 本脚本 framework 体检 + [`decision-checkpoint`](../../操作系统/07_完整工作流/decision-checkpoint.md) Q1-Q7 自检；Q1-Q7 不内嵌在本脚本里，需按起手链路单独跑。

模板根模式：当根 `README.md` 声明为“操作系统模板”且仍保留 `{{PROJECT_ROOT}}` 占位符时，体检会把项目实例专属项降级处理，例如 `Docs/1-需求文档/`、`TASKS.md`、来源项目 PROP 正文、`{{APP_REPO_DIR}}/.git/hooks`。实例化到真实项目后，占位符被替换，这些检查恢复为项目门禁。

- P4a 基础完整性
- P4b 文件大小 + 业务 P4b 治理摘要
- P4c framework 引用频率
- P4d 状态新鲜度
- P4e mount 缓存提醒
- P4f PM 切换轨迹
- P4g README / 索引一致性
- P4h 版本 / Sprint 锚点
- P4i 治理计数
- P4j-P4s 旧口径、分支策略、skills 命令新鲜度、PM 工作区入口、活跃 Markdown 链接、能力资产剩余区、治理语义锚点、hooks 配置与模板纯净度守卫

## 已知限制

关键决策点必须复核真实文件，不靠单一 bash 输出：

- Mount 缓存延迟：刚改完文件时，bash `cat` / `tail` / `wc -c` / `grep` 可能读旧缓存。
- bash `wc -c` 用于是否拆分、是否越线这类决策时，信 Windows host PS1 + Read。
- grep `.md` 被当 binary 时，用 `grep -a`。
- `[ -z ]` 判断带换行字符串容易误判，计数时用 `wc -l` 更稳。

## 输出颜色

- ✅ 通过：无需动作。
- 🟡 警戒：接近阈值、非阻塞，但要记录。
- 🔴 失败：必须先修。

## 10 类体检主题（脚本守卫为 P4a-P4s）

| # | 检查项 | 关注 |
|---|---|---|
| 1 | 源文件大小 | Cowork：<6000B 安全 / 6000-6500B 警戒 / 6500-8000B 软建议 / >=8000B 危险；Claude Code/Codex：>=8KB 建议拆职责 |
| 2 | PRD 双源一致性 | 已完成段不应残留“待 X / TBD / ⏳”等字眼 |
| 3 | PROP / ADR / RETRO 计数 | 表格计数 vs 实际目录；由 readme-index + check-os 守卫 |
| 4 | 旧路径残留 | 防 `rules/`、旧 `agent/` 当前事实源回潮 |
| 5 | 跨 session 同步状态 | Sprint 清单 vs 需求历史 vs PROP 归档 |
| 6 | 交接区健康 | 待接手 ≤3、基础段齐、已接手超期归档、状态摘要对账 |
| 7 | framework 引用频率 | 0 死代码、极低频文件评估整合；由 P4c 实现 |
| 8 | 状态.md 30 天新鲜度 | mtime + 内容日期戳；由 P4d 实现 |
| 9 | Mount 缓存陷阱提醒 | 10 次实战累计 + 4 防御规则；由 P4e 实现 |
| 10 | 模板纯净度 | 模板根不带具体项目卡；文件名和正文不回流已知来源项目关键词；由 P4s 实现 |
