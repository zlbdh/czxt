## 一、npm scripts

| 命令 | 用途 | 真实命令 |
|---|---|---|
| `npm run dev` | 本地开发（vite + 热重载） | `vite --host` |
| `npm run build` | 生产构建 → `dist/` | `vite build` |
| `npm run test` | 跑测试套件（vitest 单次运行） | `vitest run` |
| `npm run preview` | 预览生产构建 | `vite preview --host` |
| `npm run android:add` | 首次添加 Android 平台 | `cap add android` |
| `npm run android:sync` | 构建 + 同步到 Android | `npm run build && cap sync android` |
| `npm run android:open` | 打开 Android Studio | `cap open android` |
| `npm run android:run` | 构建 + 同步 + 真机/模拟器运行 | `npm run build && cap sync android && cap run android` |
| `npm run android:build-apk` | build + sync + assembleDebug（不负责归档） | `npm run build && cap sync android && cd android && gradlew.bat assembleDebug` |

## 二、APK 打包脚本（议题 BO / PROP-024 治理）

### `{{APP_REPO_DIR}}/build-apk.ps1`（主用 / Windows）

**位置**：`{{APP_REPO_DIR}}/build-apk.ps1`  
**作用**：一键打 debug APK，并复制到 `{{APP_REPO_DIR}}/apk/{{PROJECT_NAME}}-vX.X.X-debug.apk`（议题 BQ 单源）

**调用**：
```bash
cd {{APP_REPO_DIR}}
powershell -NoProfile -ExecutionPolicy Bypass -File .\build-apk.ps1
```

**议题防御**：BO（避开 `.bat` 参数解析坑）+ BQ（APK 自动归档）。

### `{{APP_REPO_DIR}}/build-apk.bat`（兼容入口 / cmd）

**位置**：`{{APP_REPO_DIR}}/build-apk.bat`  
**作用**：cmd 兼容入口，负责 build + assemble；自动归档优先用 `build-apk.ps1`。  
**议题防御**：BO（`--no-pause` 解析）+ BH（`.bat` CRLF）。

## 三、Capacitor 同步（cap sync android）

**何时跑**：每次代码改动后、装 APK 前。  
**作用**：把 `dist/` 同步到 `android/app/src/main/assets/public/`。
**关键步骤**：
1. `npm run build` → `dist/`
2. `npx cap sync android` → assets 同步
3. `cd android && gradlew.bat assembleDebug` → APK 生成
4. 复制到 `{{APP_REPO_DIR}}/apk/` 历史归档

## 六、构建相关 git 配置（议题 BG / BH 永久化）

### `{{APP_REPO_DIR}}/.gitattributes`

```
*.bat text eol=crlf      # 议题 BH 永久化
*.ps1 text eol=crlf      # 同上
*.sh text eol=lf         # POSIX 脚本保 LF
```

### `git commit -F` 强制使用（议题 BG 永久化）

**问题**：PowerShell `git commit -m "msg"` 易带 UTF-16 BOM。  
**解决**：写 `{{APP_REPO_DIR}}/commit-msg.tmp`（UTF-8 无 BOM）+ `git commit -F {{APP_REPO_DIR}}/commit-msg.tmp`。  
**实战**：议题 BG 防御 6+ 次成功。

→ 详见 [git-commit-编码规范.md](../rules/git-commit-编码规范.md)

## 九、APK 路径策略（议题 BQ v2 / 2026-05-21）

### 单源真相

**所有 APK 在 `{{APP_REPO_DIR}}/apk/`**（远端构建 + zlbdh 本地构建同一路径 / 已 gitignore）

```
{{APP_REPO_DIR}}/apk/
├── {{PROJECT_NAME}}-v3.5.8-debug.apk
├── ...
└── {{PROJECT_NAME}}-vX.Y.Z-debug.apk   ← 具体最新版本看 `{{APP_REPO_DIR}}/package.json` + `状态.md`
```

### 命名规则

```
{{PROJECT_NAME}}-v{major.minor.patch}-debug.apk      ← 标准 debug
{{PROJECT_NAME}}-v{major.minor.patch}-release.apk    ← 正式签名（A 阶段公开发布时用）
```

### 装机说明

传 APK 到手机 → 文件管理器点击安装 → 首次允许“来源不明” → 同 appId `{{APP_ID}}` 可覆盖升级；升级前先在 App 备份导出 JSON。

v2 决议：删顶级 `apk/`；历史 dev APK 可通过 `git checkout vX.X.X && build-apk.bat` 重建，版本线索见 [版本时间线.md](../../操作系统/04_台账/版本时间线.md)。单源真相收敛到 `{{APP_REPO_DIR}}/apk/`。
